# 剧本真实A测计划（第一版）

## 这条主线要解决什么
- 这条主线不是看剧本能不能吐出几场戏。
- 真正要解决的本质问题是：剧本这一层在真实试做里，到底能不能稳定产出，而且产出的东西到底是不是已经达到“好 + 稳定”。
- 如果这一步站不住，前面粗纲、人物、详细大纲再稳，也还不能说整条链真站住了。

## 本轮范围
- 只测一件事：真实生成“剧本”。
- 连续测 3 次。
- 每次都看 3 件事：
- 能不能顺利出来
- 当前质量到底好不好
- 用时大概多少

## 阶段划分
### 阶段一：切任务卡与锁测试入口
- 把当前任务正式切到这条主线。
- 锁定最短真实测试入口，只跑到剧本。
- 交付标准：能稳定开始真实测试。

### 阶段二：连续做三次真实A测
- 连续跑 3 次。
- 每次都记开始时间、结束时间、是否成功、质量结论。
- 每次测完都关闭测试软件，避免残留进程卡机器。
- 交付标准：拿到 3 份真实结果。

### 阶段三：汇总平均耗时与质量判断
- 算平均耗时。
- 判断现在剧本这一层，到底是不是“好 + 稳定”。
- 交付标准：给出不容易被推翻的真实判断。

## 当前判断
- 本质问题已经解决。
- 现在可以正式判断：剧本这一层已经在真实试做里稳定起写，且质量已经达到“好 + 稳定”。
- 支撑这个判断的证据有两层：
- 第一层：剧本入口已经真实打通，不再卡死在计划未放行。
- 第二层：修后真实主样本连续 3 次都成功落出第 1 场剧本，继续补 1 次尾巴复测后，原先那条“后续承接点稍弱”的轻微问题也已消失。
- 当前正式结论：这条“剧本真实A测主线”已完成。
