# 详细大纲真实A测计划（第一版）

## 这条主线要解决什么
- 这条主线不是看详细大纲能不能生成出 4 段字。
- 真正要解决的本质问题是：详细大纲这一层在真实试做里，到底能不能稳定产出，而且产出的东西到底是不是“真正能往下写剧本的阶段推进图”。
- 如果这一步站不住，前面粗纲和人物再好，后面剧本也还是会建立在假详纲上。

## 本轮范围
- 只测一件事：真实生成“详细大纲”。
- 连续测 3 次。
- 每次都看 3 件事：
- 能不能顺利出来
- 是不是还在抄粗纲重排
- 用时大概多少

## 阶段划分
### 阶段一：切任务卡与锁测试入口
- 把当前任务正式切到这条主线。
- 锁定最短真实测试入口，只跑到详细大纲。
- 交付标准：能稳定开始真实测试。

### 阶段二：连续做三次真实A测
- 连续跑 3 次。
- 每次都记开始时间、结束时间、是否成功、质量结论。
- 每次测完都关闭测试软件，避免残留进程卡机器。
- 交付标准：拿到 3 份真实结果。

### 阶段三：汇总平均耗时与质量判断
- 算平均耗时。
- 判断现在详细大纲这一层，到底是不是“好 + 稳定”。
- 交付标准：给出不容易被推翻的真实判断。

## 当前判断
- 本质问题已经解决。
- 现在可以正式判断：详细大纲这一层已经在真实试做里达到“好 + 稳定”。
- 当前正式结论：这条“详细大纲真实A测主线”已完成。
