# 当前主线

## 当前主线：P0 世界观/阵营/角色底账前置

**状态：新升 P0，尚未实现。**

**核心判断：** 灵感对话不能只把一句故事梗概整理出来就放行。如果用户没有提供世界观、时代/社会背景、关键阵营、场域、角色池和主线历史伤口，灵感对话 Agent 必须继续追问；没有问出来就算灵感对话失败。否则后续人物小传、剧本骨架、详细大纲只能靠模型硬编，必然出现阵营太少、人物太少、角色临时冒出、拍摄时不知道该有哪些人的问题。

**用户明确提出的产品方向：**

- 人物小传之前必须有世界观。故事需要先知道“这是一个什么世界”：朝代、现实/架空/未来、社会秩序、力量体系、历史前提、当前秩序为什么会崩。
- 世界观之后必须有阵营和场域底账。几十集项目只有一两个阵营太薄；阵营少，人物就少，戏就只能围着少数人硬转。
- 阵营之后必须有角色名册。不是所有角色都写完整小传，但所有会被拍到、会说话、会制造信息差、会承接某段戏的人，都要先在角色池里有位置。
- 核心人物写完整小传；中层人物写轻量卡；群像/功能人物只需姓名、身份、阵营、出场用途和拍摄功能。不能为了省事让后续剧本临时发明角色。
- 灵感对话 Agent 要负责把这些信息问出来。用户没说清世界、阵营、角色规模时，Agent 应追问，不应默认硬搞。
- 当前短剧/慢剧通常是一两分钟一集，60 集是常见标准长度；系统必须按 60 集规模设计世界、阵营和角色池，不能按 10-20 集小体量随便给 4 个角色。
- 角色池必须包含群像、功能人物和跑龙套人物。没有姓名的人也要有“群像位/用途位”，例如“小孩 1-8”“虾兵甲/乙”“大臣甲/乙”“寺庙僧人”“公司同事”“村民”等；他们可能只有一句台词，但剧本拍摄时必须知道这个人存在。
- 80 集项目可能出现 40-80+ 个角色位并不异常；如果系统只输出 4 个核心角色，按专业剧本生成失败处理。

**参考类比：**

- 《西游记》不是“唐僧取经 + 几个妖怪”，而是大唐、人间寺庙、天庭、西天、花果山、流沙河、高老庄、沿途国家和各路妖族山头共同构成世界与场域。
- 《鹿鼎记》不是“韦小宝冒险”，而是清初朝廷、康熙/鳌拜权力斗争、天地会、反清复明、江湖帮会和地方势力共同提供人物来源。
- 《哈利波特》《流浪地球》《三体》这类故事先成立世界规则，再生长阵营与人物。
- 《剑来》的骊珠洞天这类设定说明：一个“新手村/开局场域”本身就可能承载世界规则、历史债、气运分配、势力下注和人物命运。

### P0 新目标链路

旧链路：

```text
灵感对话 -> 创作信息总结 -> 人物小传 -> 剧本骨架 -> 详细大纲 -> 剧本
```

新链路：

```text
灵感对话 -> 创作信息总结/真源合同
  -> 世界观与背景故事
  -> 阵营/场域/权力结构
  -> 角色名册
  -> 核心人物小传
  -> 剧本骨架
  -> 详细大纲
  -> 剧本
```

### 灵感对话 Agent 新验收合同

灵感对话确认前必须尽量问出以下信息；缺失时要明确追问，不允许静默补：

1. **世界类型**
   - 现实、架空历史、未来科幻、修仙玄幻、都市、古言、律政、悬疑等。
   - 如果是现实/历史：具体时代或架空程度。
   - 如果是未来/幻想：基础规则、技术/力量体系、普通人如何生活。

2. **背景故事**
   - 这个世界在故事开始前发生过什么。
   - 哪个旧秩序、旧仇、旧制度、旧秘密正在压着主线。
   - 主角为什么会在现在被卷进去。

3. **核心资源和规则**
   - 大家争什么：权力、血脉、证据、钱、身份、土地、名声、技术、信仰、气运、婚约、案件真相等。
   - 这个资源由谁控制，违反规则会付出什么代价。

4. **阵营与场域**
   - 至少问出主要阵营、对立阵营、中立/摇摆阵营、阶段性场域。
   - 对 40 集以上项目，不能只满足 1-2 个阵营；需要主动追问更多组织、家族、公司、宗门、朝堂、地方、学校、医院、帮派、副本场域或国家/城市。

5. **角色池规模**
   - 核心人物：完整小传。
   - 中层人物：身份、阵营、立场、和主角/反派关系、出场功能。
   - 功能/群像人物：姓名或称谓、所在阵营、拍摄用途、出现段落。
   - 没有姓名的群像也必须入册：例如 `小孩甲/小孩乙`、`虾兵甲/虾兵乙`、`大臣甲/大臣乙`、`村民甲`、`保安甲`。他们不需要完整小传，但要有阵营、场景、动作/台词用途。

6. **用户不确定时的处理**
   - Agent 可以给 2-3 个方向让用户选，但必须标明这是建议，不是已确认事实。
   - 未经用户确认的世界观/阵营设定只能是草案，不能直接写进正式真源。

### 生成阶段新拆分任务

**任务 A：世界观生成/整理**

- 输入：灵感对话确认出的故事意图、用户明确设定、题材策略。
- 输出：`worldBible` 或等价结构，至少包含：
  - 世界一句话定义
  - 时代/空间/社会秩序
  - 历史前提/旧伤口
  - 力量或制度规则
  - 核心资源
  - 禁忌和代价
  - 可拍场景类型
- 验收：不得凭空把现实题材写成玄幻，也不得把架空题材写成现实公司职场。

**任务 B：阵营/场域底账生成**

- 输入：世界观、故事意图、集数、题材策略。
- 输出：`factionMatrix` 升级版，阵营不只是“主角阵营/反派阵营”，还要有：
  - 阵营名
  - 阵营定位
  - 控制资源
  - 主要人物槽位
  - 内部派系/分支
  - 与其他阵营的利益关系
  - 可承载哪些段落/场景
- 集数规模建议：
  - 10 集：至少 1-2 个阵营/场域，至少 6-10 个角色位。
  - 20 集：至少 3-5 个阵营/场域，至少 10-16 个角色位。
  - 30-40 集：至少 5-8 个阵营/场域，至少 18-28 个角色位。
  - 60 集：按常见标准长度处理，至少 8-12 个阵营/场域，至少 30-45 个角色位。
  - 80 集：至少 10-16 个阵营/场域，至少 40-60 个角色位；复杂群像项目允许到 80+ 角色位。
- 验收：40 集以上项目若只有 1-2 个阵营，按生成失败或需要追问处理。
- 角色位公式候选：
  - 最低角色位：`ceil(episodeCount / 2)`，60 集最低约 30 人，80 集最低约 40 人。
  - 标准角色位：`ceil(episodeCount * 0.65)`，60 集约 39 人，80 集约 52 人。
  - 群像/场域型项目：`episodeCount` 附近或更高，80 集出现 80+ 个角色位可以接受。
- 阵营/场域公式候选：
  - 最低阵营/场域：`ceil(episodeCount / 10)`，60 集最低约 6 个，80 集最低约 8 个。
  - 标准阵营/场域：`ceil(episodeCount / 7)`，60 集约 9 个，80 集约 12 个。
  - 玄幻/西游式游历、案件单元、地方副本、朝堂江湖并行等结构，应按标准或群像型扩张。

**任务 C：角色名册生成**

- 输入：世界观 + 阵营底账。
- 输出：完整角色池，而不是只输出核心人物。
- 角色分层：
  - `core`：主角、反派、关键情感/利益关系、阶段性强敌，必须完整小传。
  - `active`：中层执行者、盟友、内鬼、证人、权力代理人，必须轻量卡。
  - `functional`：大臣甲乙丙丁、寺庙僧人、公司同事、门房、医生、记者、路人证人等，只需用途卡。
- `crowd` / `extra`：群像、跑龙套、无名队伍、儿童群体、虾兵蟹将、士兵、村民、员工、学生、患者、记者群等，只需数量、所属场域、出场用途和是否有台词。
- 验收：后续骨架/详纲/剧本不得临时发明未入名册的承重角色；确需新增时必须回写角色名册。
- 验收：60 集项目低于 30 个角色位，80 集项目低于 40 个角色位，必须给出“为什么这个项目是极简人物结构”的解释；否则按角色池不足处理。
- 验收：每个主要场域都必须至少有主事者、执行者、信息提供者/旁观者或群像位；不能只给一个场景名，不给可拍人物。

**任务 D：核心人物小传生成**

- 输入：角色名册中的 core/active 人物。
- 输出：当前人物小传页面可见内容。
- 边界：完整小传不等于全员传记；轻量人物不能被硬扩成空泛 biography。

**任务 E：剧本骨架生成**

- 输入：世界观、阵营底账、角色名册、核心人物小传。
- 输出：骨架只能消费已确认底账，不能自造世界规则、阵营和承重人物。

### 当前明确边界

- 不要求用户一开始把所有设定都写完整；但 Agent 必须主动问。
- 不要求每个角色都写完整人物小传；但角色池必须存在。
- 不允许为了推进流程把未确认世界观、阵营名、朝代、未来规则硬写成正式事实。
- 如果用户说“你帮我定”，Agent 可以给候选方案并请求确认；确认前仍是草案。
- 这条主线优先级高于继续细修单个人物小传模板，因为没有世界和阵营底账，人物质量修得再细也会薄。
