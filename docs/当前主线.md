**本轮关键交付：**

- `CopyTextButton`：点击时构建复制文本，DOM 反馈，不给每个按钮挂 React state。
- `character-stage-copy-text.ts` / `outline-stage-copy-text.ts`：复制文本生成下沉到 model helper。
- `project-summary-cache.ts`：首页项目摘要 `localStorage` 缓存，刷新时先显旧列表。
- `switchStageSession`：切阶段先更新本地可见阶段，再等待 `apiGetProject` 补水。
- 回归测试：复制性能、人物页 motion 禁用、骨架 textarea memo、首页缓存、导航乐观切页、hook 稳定性。
- 人物生成链路：`storyIntent.protagonist` 具体姓名优先，疑似主角别名统一改回锁定名；20 集完整人物小传上限为 8，其余留在轻量卡/世界底账；兜底文案不再输出“关键杠杆/退回场外/想守住自己还能掌控”。

**验证结果：**

- ✅ `npm run typecheck`
- ✅ `npm test`：777 tests, 772 pass, 0 fail, 5 skipped
- ✅ `cd server && npm run typecheck`（本轮早前验证）
- ✅ 人物小传/骨架链路相关定向测试（本轮早前验证）
- ✅ `npx tsx --tsconfig server/tsconfig.json --test server/src/application/workspace/generate-outline-and-characters-from-confirmed-seven-questions.test.ts`

**当前边界：**

- 最新人物小传修复还没有经过用户真实项目重生测试；当前只能说工程验证通过，不能说产品质量已通过。
- 下次开工第一步：重启/确认后端加载新代码，重新生成一版人物小传，重点看唯一主角、轻量卡数量、模板话、真女主/反派大小姐是否分离。
- 人物小传质量过关后，再继续剧本骨架；骨架暂时不要先推进，避免把未确认的人物底账污染到后续链路。
- 临时骨架只是保住 UI 和人物成果，不等于高质量骨架；用户仍应检查并可重新生成骨架。
- 七问候选只保留审核价值，不能再成为人物小传/骨架生成的第二套账本。
- 首页项目列表第一次没有缓存时仍要等远端列表；后续刷新才会先显示缓存。
- 阶段导航可以先切本地 UI，但项目快照权威仍来自 `apiGetProject`。
- 前端性能继续是显性风险；改 `CharacterStage` / `OutlineStage` / 首页工作台必须实测滚动、输入、展开、复制、导航。
