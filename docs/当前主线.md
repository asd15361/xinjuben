# 当前主线

## 主题

Server Authority 迁移：剩余 IPC authority 残留收口

## 为什么现在做

- HTTP/IPC 双口径清理已完成，正式业务读写全部走 HTTP。
- 但审计发现 7 个 IPC 仍然掌握 authority（写正式状态或做放行决策）：
  1. `validateStageInputContract` — 做阶段入口放行决策
  2. `declareFormalFact` — 写 outline 正式事实
  3. `confirmFormalFact` — 修改 fact.status
  4. `removeFormalFact` — 删除 outline 正式事实
  5. `startScriptGeneration` — 写正式剧本
  6. `rewriteScriptEpisode` — 写正式剧本
  7. `executeScriptRepair` — 写正式剧本
- 另有 2 个死代码需清理：`stopScriptGeneration`、`getRuntimeConsoleState`（preload 暴露但 main 无 handler）
- 这些 IPC 违反"server 是唯一正式业务入口"原则，必须迁移到 HTTP server
