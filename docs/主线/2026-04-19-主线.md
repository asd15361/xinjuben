## 二、致命问题 1：长时间运行策略

### 2.1 问题本质

剧本生成整季（如 60 集）可能耗时数小时。HTTP 请求不能一直挂起，否则：
- 前端连接超时
- Server 资源占用
- 用户无法感知实时进度
- 无法中途暂停/恢复

### 2.2 方案对比

| 方案 | 优点 | 缺点 | 适用性 |
|------|------|------|--------|
| **轮询状态表** | 简单、可靠、无额外依赖 | 需要定时请求，延迟 1-5 秒 | ✅ 推荐（当前 PocketBase 已有 `scriptProgressBoardJson`） |
| **Server-Sent Events (SSE)** | 实时推送、单连接 | 需额外 endpoint、长连接管理 | 可考虑，但增加了复杂度 |
| **WebSocket** | 双向通信、更实时 | 需要额外服务端支持、复杂 | 不推荐（PocketBase 不原生支持） |

### 2.3 推荐方案：轮询 `scriptProgressBoardJson`

**核心设计**：

1. `POST /api/script-generation/start` 立即返回 `{ taskId, projectId, status: 'running' }`
2. 后台任务启动，每集完成时更新 `project_scripts.scriptProgressBoardJson`
3. 前端 `GET /api/projects/:projectId` 时，`ProjectSnapshotDto.scriptProgressBoard` 实时反映进度
4. 前端每 3 秒轮询一次项目快照，直到 `board.batchContext.status === 'completed' | 'failed'`

**进度板结构**（已定义）：

```typescript
interface ScriptGenerationProgressBoardDto {
  episodeStatuses: ScriptEpisodeStatusDto[]  // 每集状态
  batchContext: ScriptBatchContextDto        // 批次上下文
}

interface ScriptEpisodeStatusDto {
  episodeNo: number
  status: 'pending' | 'running' | 'completed' | 'failed' | 'skipped'
  batchIndex: number
  reason: string
}

interface ScriptBatchContextDto {
  batchSize: number
  currentBatchIndex: number
  startEpisode: number
  endEpisode: number
  status: 'idle' | 'running' | 'paused' | 'failed' | 'completed'
  resumeFromEpisode: number | null
  reason: string
  updatedAt: string  // 前端可判断是否有新进展
}
```

**前端轮询逻辑**：

```typescript
// useScriptGenerationRuntime.ts（改 HTTP 版）
async function startScriptGeneration() {
  const result = await apiClient.post('/api/script-generation/start', { projectId })
  if (!result.success) throw new Error(result.error)

  // 启动轮询
  const pollInterval = setInterval(async () => {
    const project = await apiClient.getProject(projectId)
    const board = project.scriptProgressBoard

    // 更新前端进度显示
    setProgress({
      total: board.episodeStatuses.length,
      completed: board.episodeStatuses.filter(e => e.status === 'completed').length,
      running: board.episodeStatuses.filter(e => e.status === 'running').length,
      failed: board.episodeStatuses.filter(e => e.status === 'failed').length,
      status: board.batchContext.status
    })

    // 终止条件
    if (board.batchContext.status === 'completed' || board.batchContext.status === 'failed') {
      clearInterval(pollInterval)
      // 刷新最终剧本稿
      const finalProject = await apiClient.getProject(projectId)
      setScriptDraft(finalProject.scriptDraft)
    }
  }, 3000)  // 每 3 秒轮询一次
}
```

## 三、致命问题 2：接口设计

### 3.1 需要的云端接口

| 接口 | 路径 | 功能 | 入参 | 出参 |
|------|------|------|------|------|
| **Start** | `POST /api/script-generation/start` | 启动整季生成 | `{ projectId, mode, targetEpisodes }` | `{ taskId, board, failure }` |
| **Pause** | `POST /api/script-generation/pause` | 暂停当前生成 | `{ projectId }` | `{ board, status: 'paused' }` |
| **Resume** | `POST /api/script-generation/resume` | 恢复暂停的生成 | `{ projectId }` | `{ taskId, board }` |
| **Stop** | `POST /api/script-generation/stop` | 强制停止生成 | `{ projectId }` | `{ board, status: 'stopped' }` |
| **Rewrite Episode** | `POST /api/script-generation/rewrite-episode` | 单集重写 | `{ projectId, episodeNo }` | `{ scene, failures }` |
| **Get Status** | `GET /api/projects/:projectId` | 获取实时进度（复用现有接口） | — | `ProjectSnapshotDto`（含 `scriptProgressBoard`） |

### 3.2 接口设计细节

#### 3.2.1 `POST /api/script-generation/start`

**入参**：

```typescript
interface StartScriptGenerationHttpRequest {
  projectId: string
  mode?: 'fresh_start' | 'resume'  // 默认 fresh_start
  targetEpisodes?: number  // 默认从 outline 推导
}
```

**出参**：

```typescript
interface StartScriptGenerationHttpResponse {
  success: boolean
  taskId?: string  // 后台任务 ID（用于后续 cancel）
  board?: ScriptGenerationProgressBoardDto  // 初始进度板
  failure?: ScriptGenerationFailureResolutionDto
  error?: string
}
```

**扣费策略**：

- 每集扣 1 积分（如 60 集扣 60 积分）
- 批次级扣费：每 5 集一批扣 5 积分
- 失败不扣（批次失败回滚）
- 用户余额检查前置：余额不足返回 402

#### 3.2.2 `POST /api/script-generation/pause`

**入参**：

```typescript
interface PauseScriptGenerationRequest {
  projectId: string
}
```

**出参**：

```typescript
interface PauseScriptGenerationResponse {
  success: boolean
  board?: ScriptGenerationProgressBoardDto  // 当前进度
  status?: 'paused'
  error?: string
}
```

**实现逻辑**：

- 设置后台任务 `abortController.abort()`
- 更新 `scriptProgressBoardJson.batchContext.status = 'paused'`
- 记录 `resumeFromEpisode`（当前正在生成的集数）

#### 3.2.3 `POST /api/script-generation/resume`

**入参**：

```typescript

  projectId: string
}
```

**出参**：

```typescript

  success: boolean
```

**实现逻辑**：

- 从 `scriptProgressBoardJson.batchContext.resumeFromEpisode` 开始

- 重新启动后台任务

#### 3.2.4 `POST /api/script-generation/stop`

**入参**：

```typescript
interface StopScriptGenerationRequest {
  projectId: string
}
```

**出参**：


**实现逻辑**：

- 强制终止后台任务
- 清理 `scriptProgressBoardJson`（设为 `null` 或保留最后一次进度）

## 四、致命问题 3：调度器上云

### 4.1 当前调度器位置

- `src/main/application/script-generation/runtime/run-script-generation-batch.ts`
- `src/main/application/script-generation/start-script-generation.ts`

### 4.2 调度器上云策略

**目标**：把调度逻辑从 Electron main 迁移到 Server，利用 `project_scripts` 表持久化状态。

**方案**：

1. **Server 端新增**：
   - `server/src/application/script-generation/script-generation-orchestrator.ts` — 调度器
   - `server/src/application/script-generation/batch-runner.ts` — 批次执行器
   - `server/src/api/routes/script-generation.ts` — HTTP 路由

2. **状态持久化策略**：

   - **启动时**：写入 `scriptProgressBoardJson` 初始状态
   - **每集完成时**：更新 `scriptProgressBoardJson.episodeStatuses[episodeNo].status`
   - **每批次完成时**：更新 `scriptProgressBoardJson.batchContext` + `scriptDraftJson`（批次稿）
   - **整季完成时**：写入 `scriptDraftJson`（完整稿）+ `scriptStateLedgerJson` + 清理进度板

3. **版本控制**：

   - 每次 `saveScriptState()` 时递增 `version` 字段
   - 前端轮询时可检查 `version` 是否变化，判断是否有新进展

### 4.3 状态一致性保证

**关键原则**：

1. **批次级保存**：每完成 5 集一批，立即写入 `scriptDraftJson` + `scriptProgressBoardJson`
2. **单集级更新**：每集完成时，更新 `episodeStatuses[episodeNo]`，前端可实时看到
3. **失败时保存**：批次失败时，保留已生成部分 + `scriptFailureResolutionJson`
4. **暂停时保存**：暂停时，保存当前进度 + `resumeFromEpisode`

**实现示例**：

```typescript
// server/src/application/script-generation/batch-runner.ts
async function runBatch(input: BatchInput): Promise<void> {
  const { projectId, startEpisode, endEpisode } = input

  // 更新进度板：批次开始
  await updateProgressBoard(projectId, {
    batchContext: { status: 'running', currentBatchIndex: ..., updatedAt: new Date().toISOString() }
  })

  for (let episodeNo = startEpisode; episodeNo <= endEpisode; episodeNo++) {
    // 更新进度板：当前集开始
    await updateProgressBoard(projectId, {
      episodeStatuses: { [episodeNo]: { status: 'running', batchIndex: ... } }
    })

    try {
      const scene = await generateEpisode(episodeNo, input)

      // 更新进度板：当前集完成
      await updateProgressBoard(projectId, {
        episodeStatuses: { [episodeNo]: { status: 'completed', batchIndex: ... } }
      })

      // 批次级保存：累积稿
      await appendScriptDraft(projectId, scene)

    } catch (error) {
      // 更新进度板：当前集失败
      await updateProgressBoard(projectId, {
        episodeStatuses: { [episodeNo]: { status: 'failed', reason: error.message } }
      })

      // 批次级保存：失败信息
      await updateFailureResolution(projectId, { kind: 'failed', reason: error.message })

      break  // 停止批次
    }
  }

  // 批次完成，更新状态
  await updateProgressBoard(projectId, {
    batchContext: { status: 'completed', updatedAt: new Date().toISOString() }
  })
}
```

### 4.4 前端状态同步

**轮询策略**：

1. 每 3 秒调用 `GET /api/projects/:projectId`
2. 解析 `scriptProgressBoard`：
   - 计算 `completed / total` 百分比
   - 显示当前正在生成的集数
   - 失败时显示失败原因
3. `version` 变化时触发 UI 更新（避免重复渲染）

**Store 层**：

```typescript

  scriptProgressBoard: ScriptGenerationProgressBoardDto | null

  setScriptProgressBoard(board: ScriptGenerationProgressBoardDto): void
}
```

## 七、风险区

1. **长时间任务资源占用**：Server 端需合理管理并发任务数
2. **扣费策略复杂**：批次级扣费 + 失败回滚 + 单集重写扣费
3. **状态一致性**：批次失败时保留已生成部分，需精细处理
4. **前端轮询频率**：过高会浪费资源，过低体验差（推荐 3 秒）
