# 2026-04-18 主线

这份文档当前只服务这一条主线：

把“前后端分离半拉子工程”收口成唯一正式工程主链：`server/http api -> server/application -> PocketBase`，并逐阶段清退旧 `main/ipc + 本地 project-store`。

## 当前主线在解决什么

这条主线解决的是“HTTP 已经接进来了，但正式业务入口、正式存储入口、旧 IPC 壳还在并存”的问题：

1. 前端现在仍同时走 HTTP 和 `window.api`。
2. HTTP 粗纲/人物当前还是简化旁路，不是正式主链等价迁移。
3. 项目真相还在本地文件存储，没有统一落到 PocketBase。
4. `src/main/ipc` 和 `project-store` 仍在长期冒充正式 authority。

## 为什么现在要做这条主线

- 当前最大风险已经不是“有没有 API”，而是“双入口 + 双存储 + 双 authority”。
- 如果现在不先把工程主权切死，后面阶段 2、3 的数据建模和仓储迁移会继续建立在摇摆口径上。
- 用户已经明确要求三条铁律：
  - HTTP 粗纲/人物必须 100% 对齐原 Electron 主链
  - 项目数据必须云端落 PocketBase
  - HTTP 对齐后，对应 IPC 必须物理退场

## 当前补充状态（2026-04-18 登录弹窗 UI 排障）

- 当前新增事实：
  - 登录弹窗“被挡在页面后面、位置上飘”的问题，已经定位到 renderer 层前端挂载方式。
  - 原来的 `LoginModal` 虽然写了 `fixed + zIndex`，但它挂在 `AppHeader` 组件树里，而 `AppHeader` 本身带有 `backdrop-blur-sm` 和 `relative z-10`，会形成局部 stacking context。
  - 这会导致登录弹窗被 header 这层局部上下文污染，表现成“掉到后面、半透明可见、位置错位”。
- 当前已执行动作：
  - `src/renderer/src/components/LoginModal.tsx` 已改成 `createPortal(..., document.body)`。
  - 已补 `document.body.style.overflow = 'hidden'` 的滚动锁，避免打开登录框时页面继续滚动造成视觉偏移。
  - 根仓 `npm run build` 已通过，确认这次前端层级修复没有打坏编译。
- 当前仍待确认：
  - 需要在用户重启电脑并重新打开 Electron 后，复测登录弹窗是否真正保持顶层、居中、不再错位。
  - 如果复测后仍有错位，再继续查根布局容器是否还存在 `transform / filter / perspective / overflow` 之类的二次污染源。
