## 当前判断

1. 当前仓库的本质问题已经不是“有没有 HTTP”，而是“HTTP、PocketBase、本地 IPC 三套权力还没收成一套”。
2. 阶段 2 现在已经把 PocketBase 数据骨架钉住了；后面阶段 3 可以直接围绕这 6 张表做 Repository，不用再边写边猜存储结构。
3. 当前正式项目真相已明确走“主表 + 分阶段子表 + 每表独立 version”的方案，不再沿着本地 `projects.json` 那种整包快照思路继续长。
4. 阶段 3 已把“云端怎么读、怎么写、怎么拼快照”落到服务端；阶段 4 已开始把前端项目 CRUD 和工作台初始化读链切到 HTTP。
5. 当前阶段 5 已把七问“确认并保存”这条前置真相链收成 HTTP 单线；七问确认后，前端已开始只认服务端快照，不再自己本地盲改再补一次 IPC。
6. 当前还没动粗纲/人物正式生成链；这轮只确认“七问作为前置弹药”已经变成纯云端真相。
7. 现在的验收第一优先级不再是单个页面能不能跑，而是：正式业务入口、正式存储入口、旧业务壳，三者边界有没有写死。
