## 当前判断

1. 七问现在已经是独立 stage，不再和聊天页绑在一个面板里。
2. 用户现在可以先拿到“七问保存成功”的确定性，再决定是否启动粗纲/人物长任务。
3. Stage 3 当前总预算已经扩到 `10` 分钟；如果后面还慢，下一刀该继续查真实后端耗时，而不是再回头怪 UI 绑死。
4. 这轮 20 集“读秒结束交白卷”的主因已经从“粗纲空返回”进一步收缩成“势力矩阵 120 秒超时 + 前台错误不够直白”。
5. 现在只要 `faction_matrix` 超时，前台应该直接看到明确红字，而不是再看到倒计时结束后空白。
1. 当前 Stage 3 的人物链主病灶已经从“单次超大 JSON 调用”被正式改成“势力分块生成”；后续如果还慢，优先继续看粗纲阶段，不再先怀疑人物 V2 还是老单包模式。
2. `character_profile_v2_parse_failed` 现在不会再以“空白白板”方式出现；失败时应直接带具体势力名。
1. `character_profile_v2_parse_failed` 这次不是前端假提示，也不是上一次 `faction_matrix_parse_failed` 残留；是人物 V2 Agent 新炸点。
2. 当前主问题已从“到底是不是新报错”收缩到“人物 V2 输出经常缺字段/截断，而 agent 原来没有诊断和纠偏能力”。
3. 这轮代码已把人物 V2 解析链补到和 `faction-matrix-agent` 同等级别，但还没做新的真实前台复跑。
1. 根目录现在已经回到“只留正式入口、配置和项目级说明”的状态，不再混着一次性排障脚本。
2. `tools/e2e` 现在至少在物理目录上已经分清 runner、utils、tests、prototype、analysis，不再把所有东西混在一层根目录里。
3. 这轮剩余风险不是“根目录还脏”，而是迁移后的 `tools/e2e` 脚本还需要继续做运行级复核；当前只做了路径和结构收口，没有整批重跑所有脚本。
