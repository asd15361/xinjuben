## 正式主链

```text
renderer -> server/http api -> server/application -> src/shared -> PocketBase（轻量 metadata）
  |
  v
本地 content store（剧本正文、运行时状态真相源）
```

- `src/renderer/`：前台交互和展示，收到 server 结果后写本地内容真相源
- `server/src/api/`：正式 HTTP 路由，只鉴权/扣积分/AI 调用并返回结果，不再写完整剧本大 JSON 到 PB
- `server/src/application/`：核心业务和生成逻辑
- `src/shared/`：前后端唯一共享合同与领域逻辑
- `server/src/infrastructure/pocketbase/`：账户与计费真相源（账号、登录、积分、订单、权限、项目轻量索引）
- `src/main/infrastructure/storage/local-content-store.ts`：本地内容真相源（剧本正文、草稿、运行时状态）

创作工序命名：
```text
灵感对话 -> 创作信息总结/故事梗概 -> 人物小传/底账 -> 剧本骨架 -> 详细大纲 -> 剧本
```

- 用户可见统一叫“剧本骨架”，不再叫“粗略大纲”
- 内部 `rough_outline` 类型名可暂时保留，但不得新增用户可见“粗略大纲”文案
- 七问候选 session 是草稿态和审核辅助；人物小传/剧本骨架当前由 `storyIntent` 直驱，旧锁定七问只作为兼容约束注入

## 当前 owner

- HTTP API：`server/`
- 共享合同：`src/shared/`
- 桌面壳：`src/main/`（只保留 IPC 事件桥接、导出、系统信息）
- 剧本生成 authority：`server/src/application/script-generation/`
- 剧本审计（只读辅助计算）：`src/main/application/script-generation/audit/`（当前仍是本地辅助链，不是正式写入口）
- 七问生成：`server/src/api/routes/generate.ts`（HTTP `/api/generate/seven-questions`）
- 剧本修稿执行 authority：`server/src/application/script-generation/repair/`（已补实，通过 HTTP `/api/script-audit/execute-repair` 调用）

## 人物小传/剧本骨架链路

```text
storyIntent
  -> faction matrix / character profiles
  -> rough outline bundle
  -> outlineDraft + characterDrafts + entityStore
```

- 人物小传先于剧本骨架生成，骨架批次失败不能丢人物小传。
- `rough_outline_batch_parse_failed` 走可见临时骨架恢复，返回 `outlineGenerationError` 给前端提示。
- 20 集项目的势力矩阵走轻量编制；60 集以上再使用完整多势力/多分支/多人物要求。
- 七问不是这条链路的第二账本；如果旧项目已经锁过七问，只把它折叠成兼容约束。

## 学习型创作资产层

写作逻辑分三层：
```text
A Stable Writing Kernel（稳定内核）
B Market Playbook（市场打法包）
C Project Intent（项目自身设定）
```

- A 放稳定合同：平台安全、剧本格式、七问、剧本骨架、连续性、每集施压/反击/钩子
- B 放市场变化：优秀剧本样本提炼出的开局、爽点、钩子、反派压迫、主角反击、道具信息承载
- C 放用户项目：题材、受众、垂类、人物、集数、故事梗概、锁定七问
- MarketPlaybook 必须版本化、可审核、可选择；不能覆盖稳定内核
- 导入优秀剧本时先提取打法包草案，不长期把全文塞进 Prompt

MarketPlaybook Prompt 注入顺序（P8）：
```text
A Stable Kernel（稳定内核：格式、安全、七问合同、骨架合同）
  ↓
B MarketPlaybook（市场打法包：阶段特定 patterns / antiPatterns / promptRules）
  ↓
C Project Intent（项目设定：题材、受众、人物、集数、梗概）
```

注入位置：
- 七问：`seven-questions-agent.ts` → `buildSevenQuestionsPrompt` 内，关键人物之后、七问格式说明之前
- 剧本骨架：`rough-outline-stage-prompts.ts` → overview + batch，MarketProfile 之后
- 正式剧本：`create-script-generation-prompt.ts` → MarketProfile 之后

阶段特定输出：
- `seven_questions`：opening_pressure, payoff, villain_pressure, hook
- `script_skeleton`：payoff, protagonist_action, villain_pressure
- `episode_script`：opening_pressure, villain_pressure, protagonist_action, payoff, hook, dialogue_style

MarketPlaybook 低成本闭环验证（P10）：
```text
source sample
  → createMarketPlaybookDraftFromSamples（只生成 draft）
  → 人工审核/改写 promptInstruction + qualitySignal
  → validateMarketPlaybookBeforeActivation
  → activateMarketPlaybookDraft
  → buildMarketPlaybookPromptBlock（进入 Prompt）
  → inspectPlaybookAlignment（只观测，不改 overallScore，不触发修稿）
```

MarketPlaybook 审核工作台边界（P11）：
- 当前能从首页生成、编辑、预览并复制 active playbook JSON
- 当前不写数据库，不写全局 registry，不自动影响新项目生成
- 后续若要真正启用用户自定义打法包，必须补“active playbook 持久化 + server 解析自定义 playbookId”
