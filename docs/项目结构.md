# 项目结构

这份文档让人看一遍就知道仓库的正式线、分层职责和出问题先查哪里。

详细代码结构图谱见 `graphify-out/graph.html`，那里有完整的代码节点和依赖关系。

## 业务总链

```
用户输入 → 确认信息 → 七问篇章 → storyIntent → 粗纲 → 人物小传 → 详细大纲 → 剧本
```

产品页签：`chat` → `seven_questions` → `outline` → `character` → `detailed_outline` → `script`

## 工程总链

```
renderer → server/http api → server/application → shared/contracts + shared/domain → PocketBase → 返回项目快照 → renderer hydrate
```

分层边界：
- `shared` — 真相和共享合同
- `server` — 正式业务、生成、持久化
- `PocketBase` — 正式项目真相
- `main/preload` — 桌面壳和系统桥接（待退场）
- `renderer` — 页面、交互、展示

## 仓库地图

| 目录 | 职责 | 身份 |
|------|------|------|
| `AGENTS.md` | 仓库最高口径 | 现役 |
| `src/shared/` | 跨层合同、真相源、共享领域逻辑（前后端唯一共享包） | 现役真相层 |
| `server/` | 唯一正式业务入口（CRUD/生成/积分/认证），通过 @shared 别名引用 src/shared | 现役主链 |
| `server/src/infrastructure/pocketbase/` | 正式项目仓储落点 | 现役存储层 |
| `server/src/api/routes/` | HTTP 路由（auth/credits/generate/outline-characters/projects/scripts） | 现役入口 |
| `server/src/application/` | 业务服务层 | 现役 |
| `src/main/` | Electron 桌面壳（7 文件，系统桥接+导出） | 待退场 |
| `src/preload/` | 桌面壳桥接（2 模块：system+workspace） | 待退场 |
| `src/renderer/` | 前台工作台，按 features 分目录 | 现役 |
| `server/pb_migrations/` | PocketBase 表结构迁移 | 现役 |
| `tools/e2e/` | 验收、观察、原型脚本和证据 | 观察口 |
| `docs/` | 现役文档 + 归档 | 5 份现役 + 归档区 |

## 正式主链各层

### 用户输入与确认信息线
- 正式真相：`storyIntent`、`generationBriefText`、角色名单
- 出问题先查：`server/src/application/`、`project-repository.ts`

### 七问确认线
- 正式真相：`outlineDraft.outlineBlocks[].sevenQuestions`
- 出问题先查：`save-confirmed-seven-questions.ts`、`seven-questions-authority.ts`

### 粗纲线
- 正式真相：`outlineDraft`、`outlineBlocks`
- 出问题先查：`faction-matrix-agent.ts`、`useOutlineCharacterGeneration.ts`

### 人物线
- 正式真相：`entityStore`、`characterDrafts`
- 出问题先查：`project-snapshot-normalize.ts`、`derive-character-stage-sections.ts`

### 详细大纲线
- 正式真相：`detailedOutlineSegments`、`episodeBeats[]`、`episodeControlCard`
- HTTP：`POST /api/generate/detailed-outline`，扣 5 积分
- 出问题先查：`detailed-outline-entry-guard.ts`、`episode-control-agent.ts`

### 剧本线
- 正式真相：`scriptDraft`、`scriptControlPackage`
- HTTP：Start/Pause/Resume/Stop/Rewrite
- 出问题先查：`start-script-generation.ts`、`create-script-generation-prompt.ts`

## 出问题先查

| 现象 | 先查 |
|------|------|
| 项目 CRUD 不对 | `projects.ts` 路由、`api-client.ts`、`project-repository.ts` |
| 七问没落盘 | `seven-questions-authority.ts`、`project-repository.ts` |
| 生成超时/白板 | `resolve-ai-stage-timeout.ts`、对应 agent |
| 详纲被门禁拦 | `detailed-outline-entry-guard.ts`、`validate-stage-input.ts` |
| 剧本 pending | `start-script-generation.ts`、进度板持久化 |
| HTTP 404 | `server/src/api/routes/` 路由注册/路径 |
| 积分异常 | `credit-service.ts`、路由中间件 |
| 认证失败 | `api-client.ts`、JWT |

## 高风险区

- `create-script-generation-prompt.ts` — 容易重新长胖
- `compact-script-context.ts` — 容易压掉有用控制块
- `character-contract.ts` — 残留旧口径
- `build-ledger-prompt-blocks.ts` — 容易灌回正文级上下文

## 维护规则

1. 只写总图、职责、问题定位口，不写流水账
2. 正式主链变了就更新
3. 原型线/历史壳必须标身份
4. 新目录进现役就补进地图
5. 代码结构详情看图谱 `graphify-out/graph.html`
