## 正式主链

```text
renderer -> server/http api -> server/application -> src/shared -> PocketBase（轻量 metadata）
  |
  v
本地 content store（剧本正文、运行时状态真相源）
```

- `src/renderer/`：前台交互和展示，收到 server 结果后写本地内容真相源
- `server/src/api/`：正式 HTTP 路由，只鉴权/扣积分/AI 调用并返回结果，不再写完整剧本大 JSON 到 PB
- `server/src/application/`：核心业务和生成逻辑
- `src/shared/`：前后端唯一共享合同与领域逻辑
- `server/src/infrastructure/pocketbase/`：账户与计费真相源（账号、登录、积分、订单、权限、项目轻量索引）
- `src/main/infrastructure/storage/local-content-store.ts`：本地内容真相源（剧本正文、草稿、运行时状态）

## 当前 owner

- HTTP API：`server/`
- 共享合同：`src/shared/`
- 桌面壳：`src/main/`（只保留 IPC 事件桥接、导出、系统信息）
- 剧本生成 authority：`server/src/application/script-generation/`
- 剧本审计（只读辅助计算）：`src/main/application/script-generation/audit/`（当前仍是本地辅助链，不是正式写入口）
- 七问生成：`server/src/api/routes/generate.ts`（HTTP `/api/generate/seven-questions`）
- 剧本修稿执行 authority：`server/src/application/script-generation/repair/`（已补实，通过 HTTP `/api/script-audit/execute-repair` 调用）

## 内容质量检测系统

```
screenplay-content-quality.ts（共享层）
├── 循环检测（loopPatterns）
├── 人物弧线检测（characterArcDetection）
├── 主题锚定检测（computeThemeAnchoring）
├── 情节新鲜度检测（computePlotNovelty）
├── 戏剧推进检测（computeDramaticTurn）
├── 场次引擎检测（computeSceneEngine）
├── 角色功能检测（computeCharacterFunction）
├── 商业评分检测（openingShock/villainOppression/catharsisPayoff/punchlineDensity/hookRetention）
├── 信息密度检测（computeInformationDensityScore）← P4
├── 剧本格式检测（computeScreenplayFormatScore）← P4
├── 垂类质量检测（computeMarketQuality）← P4
│   ├── 男频维度（statusReversal/powerProgression/hiddenCard/publicPayoff/villainHierarchy）
│   └── 女频维度（emotionalIdentification/relationshipTension/powerBorrowing/supportingPowerReveal/femaleGrowth）
├── 连续性检测（storyContinuityScore）← P4 snapshot 驱动
│   └── inspectStoryContinuityAgainstSnapshot（5 类：主角状态/道具/钩子/反派递进/硬约束）
├── buildContentRepairSignals（低分→结构化修稿信号）← P5
├── inspectContentQualityEpisode（单集检测入口，接受 marketProfile / snapshot）
└── inspectContentQualityBatch（批量检测入口，含均分统计，接受 snapshots[]）
```

故事状态快照系统（共享层）← P4：
```
story-state-snapshot.ts
├── buildStoryStateSnapshot（从 outline/characters/intent/ledger 提取结构化快照）
└── buildStoryStateSnapshotPromptBlock（渲染为 prompt 文本块）

StoryStateSnapshotDto（contracts/story-state.ts）
├── protagonistState（statusSummary + emotionalArc）
├── antagonistState（statusSummary + threatLevel + currentGoal）
├── relationshipState（keyRelationship + currentTension）
├── activeProps（当前道具列表）
├── unresolvedHooks（未兑现钩子，最多 5 条）
├── activeForeshadowing（活跃伏笔，最多 5 条）
├── continuityConstraints（硬约束，最多 8 条）
└── previousEpisodeEnding（上一集落点）
```

内容质量 UI 面板（renderer）← P6：
```
ScriptQualityReportPanel.tsx
├── 市场定位（marketProfile.audienceLane + subgenre）
├── 质量概览（marketQualityScore / informationDensityScore / screenplayFormatScore / storyContinuityScore）
├── 商业传播力（openingShock / punchlineDensity / catharsisPayoff / villainOppression / hookRetention）
├── 返修建议（weakEpisodes + postflight issues）
└── ScoreBar 色标：>=75 绿色 / 60-74 黄色 / <60 红色
```
数据流：`server API → useWorkflowStore.scriptStateLedger → ScriptStage props → ScriptQualityReportPanel`

回归测试 ← P6：
```
server/scripts/
├── regression-test-male-urban.ts（男频都市逆袭 20 集，含 marketProfile）
└── regression-test-female-ceo.ts（女频霸总甜宠 20 集，含 marketProfile）

```
run-script-generation-batch.ts
├── buildContentRepairPrompt（问题清单格式修稿提示词）← P5 重构
