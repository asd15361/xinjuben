# 第一板块到自动生成链收口（第一版）

## 本轮要压的本质问题
- 粗纲不能再是总简介后拆，必须天然按集生成。
- 人物小传不能再只是说明书，必须直接给出人物发动机。
- 聊天、粗纲、人物、详纲、剧本、生成中状态，必须都挂到同一个项目档案里。
- 自动生成链不能再依赖“用户刚好点进那个页面”，必须项目级持续推进。

## 本轮执行路径
1. 先改公共数据结构：粗纲分集、人物抓手、项目级生成状态。
2. 再改第一板块生成端：直接产出 10 集，不再后拆。
3. 再改前台承接：粗纲按集编辑，人物按抓手展示和编辑。
4. 再改项目级自动链：详纲和剧本不再依赖当前页面。
5. 最后补旧项目兼容、短测脚本、编译和短测验收。

## 本轮验收标准
- 粗纲默认就是 10 个框，每集单独可改。
- 粗纲框随文字自动长高，不出现框内上下滚动。
- 人物可以新增、删除，不再残留删不掉的新人物。
- 人物页先看到抓手，再看到说明。
- 切页面后，生成状态和倒计时还在。
- 聊天、粗纲、人物、详纲、剧本都能持久化。
- 详纲和剧本能在项目级自动继续往下走。
