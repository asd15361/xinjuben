# 第五阶段第一刀收口判断（第一版）

## 这份判断只回答一件事

第五阶段第一刀，也就是“先收首页体验层级和入口顺手度”这一刀，现在能不能正式收口。

## 先说结论

可以正式收口。

## 为什么可以收口

第一刀原来要解决的，不是“把首页做得更花”，而是：

- 让用户一进来就知道先干什么
- 让首页入口主区和项目列表区不再挤成一团
- 让首页页面文件回到总装职责，不再自己同时管所有动作和展示

现在这件事已经做到第一刀该做到的深度。

已经完成的关键动作有四处：

- `HomePage.tsx` 回到页面总装职责
- `useHomePageActions.ts` 接走入口动作和状态
- `HomeHeroPanel.tsx` 接走首页主入口区
- `ProjectListPanel.tsx` 接走项目列表区

这说明第一刀最核心的本质问题已经被解决：用户进入首页时，先看到什么、先做什么、项目列表在什么位置，层级已经比原来清楚。

## 为什么现在不继续留在第一刀里反复拆

因为第一刀的目标，是先把“首页入口看起来乱、重心不清”这件事压住，不是把首页无限细拆。

如果现在继续停在第一刀里反复拆，会开始出现两个偏差：

- 容易把“入口顺手度”做成“机械继续切块”
- 容易为了局部更细，拖住下一处更前面的体验阻断

这就不再是在解决当前最前面的本质问题。

## 第一刀的正式判断

第五阶段第一刀现在的正式判断是：

- 本质问题：已解决到这一刀要求
- 首页入口层级是否已回到更清楚的状态：已回到
- 是否还需要继续停留在第一刀：不需要
- 是否允许切入第二刀：允许

## 第二刀现在为什么轮到它

因为首页这个第一触点已经先压住了。

接下来更前面的体验问题，已经变成详细大纲页的输入负担：

- `DetailedOutlineStage.tsx`

它现在的问题不是功能缺，而是：

- 输入区一口气全展开
- 提示、校验、阶段合同、事实锚点同时出现
- 用户会觉得信息很多，但不容易知道先填哪

所以现在切到第二刀，才是在继续解决当前更前面的本质问题。

## 这次收口后主线怎么变

从现在开始：

- 第五阶段第一刀视为完成
- 当前小任务切到第二刀
- 第二刀主拆 `DetailedOutlineStage.tsx`
- 目标是把输入负担、阅读顺序和提示层级收清楚
