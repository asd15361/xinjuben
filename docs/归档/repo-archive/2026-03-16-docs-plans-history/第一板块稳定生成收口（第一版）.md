# 第一板块稳定生成收口（第一版）

## 这轮只打一个本质
- 不是先追粗纲写得漂不漂亮。
- 不是先追人物小传狠不狠。
- 先把第一板块“稳定生成”做实。
- 验收标准只有一个：连续短测里，粗纲和人物能稳定落地，不再整包报废。

## 当前已经确认的本质问题
- 第一板块现在一次性要同时交付太多东西：故事意图、十集粗纲、粗纲摘要、候选事实、人物小传、人物抓手、目标、短板、弧光。
- 交付内容太重，但最后收口方式太死，只要最后格式有一点歪，就整包失败。
- 聊天阶段和收束阶段的口径没有完全统一：前面还在追问，后面又要求直接定稿，导致系统经常卡在“信息够一点、又不够到底”的中间层。
- 所以当前最前面的真问题不是“内容差”，而是“第一板块真实生成不稳定”。

## 执行路径
### 第一刀：减负
- 第一板块改成两段式，不再一口气吐完整包。
- 先稳定产出：故事意图 + 十集粗纲 + 最少必要事实。
- 再根据已落下来的粗纲，单独生成人物小传和人物抓手。
- 原则：一段只做一件大事，先保落地，再保丰满。

### 第二刀：收口改成分层接住
- 不能再让一个字段写歪，就把整包全部判死。
- 新规则改成：
- 先接住粗纲主骨架。
- 再接住人物主骨架。
- 候选事实和细节字段如果有问题，只影响自己，不拖死整包。
- 原则：哪块坏修哪块，不再全盘归零。

### 第三刀：失败后不许全空
- 就算生成不完整，也要先把已经成功的粗纲或人物接到页面和项目档案里。
- 让用户至少看到第一版草稿，而不是只看到“这次收束没有成功”。
- 原则：失败可续，不许归零。

### 第四刀：聊天和收束口径统一
- 聊天阶段负责把信息聊到“够开工”。
- 一旦够开工，就直接按已有信息收第一版，不再继续一边追问、一边拦住不让生成。
- 缺的内容标成待补，不再作为整包失败理由。
- 原则：先出第一版，再继续追精度。

### 第五刀：专项短测
- 只测第一板块，不跑长链。
- 每次测试都只看：
- 能不能稳定生成。
- 失败时会不会整包归零。
- 粗纲是不是十个框。
- 人物是不是能真正落下来。
- 连续短测通过后，才进入下一轮“内容质量升级”。

## 这轮验收标准
- 连续短测至少两到三次，第一板块都能稳定落地。
- 不再出现“一点格式歪就整包失败”。
- 就算有局部异常，也能先看到第一版粗纲和人物。
- 第一板块过线后，才允许重新讨论粗纲质量和人物质量。
