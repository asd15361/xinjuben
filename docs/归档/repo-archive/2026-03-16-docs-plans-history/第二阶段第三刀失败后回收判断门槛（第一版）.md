# 第二阶段第三刀失败后回收判断门槛（第一版）

## 这份文档只回答一件事

系统失败过一次、或者中途断过一次之后，到底什么情况还能继续往下走，什么情况必须停下来重看。

## 先说结论

第二阶段第三刀只认 3 档判断：

1. 可以继续
2. 可以恢复后继续
3. 必须停下重看

以后不再允许用这些模糊说法：

- “看起来问题不大”
- “大概可以继续”
- “先试试看再说”

## 为什么这一刀现在必须做

因为现在项目已经证明了：

- 失败链路能跑
- 续跑链路能跑
- 恢复之后系统还能重新回到主链

但这还不够。

现在还缺的是：

**恢复之后到底还值不值得继续信。**

这才是这一步的本质问题。

## 这一刀真正要解决的本质问题

不是“会不会救火”。

而是：

**救完火之后，系统是不是还能继续认账。**

## 三档正式判断

### 第一档：可以继续

满足下面 4 条，才算“可以继续”：

1. 主链状态已经恢复正常
2. 当前项目还认得出自己是谁，没有丢项目、丢阶段、丢已确认事实
3. 当前阶段的关键门禁还能正常工作
4. 恢复后没有出现新的明显失真

大白话说：

- 系统不只是重新亮起来了
- 而是真的还能接着干原来的事

### 第二档：可以恢复后继续

满足下面 3 条，才算“可以恢复后继续”：

1. 当前失败是单次生成失败、单次续跑失败、或单次读取抖动
2. 项目身份、已确认事实、主链入口都还在
3. 恢复后回到主链，再跑一轮能重新过关键门禁

大白话说：

- 这类问题不是根烂了
- 更像跑的时候绊了一下
- 重新站起来还能接着走

### 第三档：必须停下重看

只要出现下面任意一种，就必须停：

1. 项目身份丢了，系统已经认不出当前项目
2. 已确认事实丢了，或下游门禁已经不再认账
3. 恢复后主链门禁还在持续失效
4. 恢复后虽然能点，但结果已经明显失真

大白话说：

- 这类不是“先继续再说”
- 而是“继续走只会把错越带越深”

## 当前正式门槛

### 什么情况算“恢复成功”

恢复成功不是指页面重新打开。

恢复成功必须同时满足：

1. 能重新进入原项目
2. 能重新回到原工序
3. 关键锚点还在
4. 关键门禁还能重新过

### 什么情况算“只是表面恢复”

下面这些都不算真正恢复：

- 页面打开了，但项目找不到
- 项目还在，但已确认事实掉了
- 可以继续点按钮，但关键门禁已经不认账
- 可以继续生成，但结果已经跟前面真相脱线

## 当前这条门槛解决了什么本质问题

这一步真正解决的是：

**以后再遇到失败或续跑，不会再只看“能不能重新点起来”，而是会看“恢复后还能不能继续信”。**

这件事不容易再被推翻。

## 这一刀之后怎么接

这一刀之后，第二阶段只剩最后一步：

**做第二阶段总回看。**

总回看要回答的就是：

- 质量承诺到底有没有比第二阶段开始时更硬
- 如果还没到位，最前面的剩余口子是什么

## 最后结论

第二阶段第三刀当前正式口径就是：

**失败后不是能亮起来就算恢复，只有“项目还认账、事实还认账、门禁还认账、结果没失真”才算真的可以继续。**
