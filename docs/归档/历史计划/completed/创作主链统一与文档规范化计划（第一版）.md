# 创作主链统一与文档规范化计划（第一版）

更新时间：2026-03-20

## 一句话结论

先不继续瞎放量。

这轮先把 `粗纲 -> 人物小传 -> 详细大纲 -> 剧本` 四层工序统一成唯一真相，再把这套真相写进正式文档、任务卡和代码。只有这样，后面的 `10 / 20 / 30 / 40 / 50 / 60` 测试结果才算正式结果。

## 这轮到底在解决什么

当前真正的问题不是“某一层偶尔写不好”。

真正的问题是这 4 件事还没有完全统一：

- 工序单位不统一
- 喂法不统一
- 文档口径不统一
- 代码里还有旧口径残留

只要这 4 件事没收死，继续放量测出来的结果都不稳。

## 统一后的唯一流程

### 1. 正式底稿层

- 只负责把聊天整理成正式创作底稿
- 不提前替下游做创作判断
- 不抢粗纲、人物、详纲、剧本的定义权

### 2. 粗略大纲层

- 先给出整季总方向
- 再按 `10 集` 一个规划块生成
- 每个规划块都必须有：
  - `blockNo`
  - `startEpisode`
  - `endEpisode`
  - `summary`
  - `episodes`

统一判断：

- `10 集` 是规划单位
- 不是整季一次性直接把所有集数当成一个块塞下去

### 3. 人物小传层

- 不再一口气只吐“一个整包人物”
- 改成双层：
  - `coreCharacters`
  - `activeCharactersByBlock`

统一判断：

- 核心角色包负责整季长期有效人物
- 活跃角色包负责当前 `10 集` 规划块真正会上场的人
- 路人和功能角不抢核心角色窗口

### 4. 详细大纲层

- 跟着粗纲块走
- 默认 `10 集` 一个详纲块
- 每个详纲块内部再拆若干段
- 段数由块内集数决定，不再固定死成四大段

统一判断：

- 逐集细纲才是剧本层唯一真源
- “四段概览”如果要存在，只能是派生视图，不能反过来掌权

### 5. 剧本层

- `5 集` 为唯一写作批次
- 批次里的每一集可以单独写
- 但每一集都必须吃同一份 `5 集批次上下文`

当前集唯一输入只认这 4 样：

- 当前 `5 集` 批次前情
- 当前集对应的逐集细纲
- 当前块活跃人物
- 正式剧本输出合同

统一判断：

- runtime 只做轻量调度
- runtime 不能反过来改业务工序
- 不能再出现“调度像 5 集，写作又散回单集各吃各的旧上下文”

## 这轮收死的唯一真相

### 1. 规划真相

- `10 集` 是规划单位
- 粗纲和详纲都围绕 `10 集块` 组织

### 2. 写作真相

- `5 集` 是剧本写作单位
- 剧本批次不再被 runtime 临时改口

### 3. 人物真相

- 只有“核心角色 + 当前块活跃角色”这一个正式表达
- 不再保留“整季人物包就是唯一人物口径”的旧脑子

### 4. 细纲真相

- 逐集细纲是剧本当前集唯一真源
- 不再允许一处按 `segments` 拿、一处按别的字段拿

### 5. 剧本真相

- 最终结果只认正式剧本正文 `screenplay`
- `Action / Dialogue / Emotion` 只保留为内部摘要

## 文档必须统一写清的 5 件事

### 1. 什么是 `10 集规划单位`

- 一个规划块默认覆盖 10 集
- 它负责这 10 集的共同方向、阶段压力和分集骨架

### 2. 什么是 `5 集剧本单位`

- 一个剧本批次默认覆盖 5 集
- 它负责这 5 集共享的前情、压力线和活跃人物窗口

### 3. 什么是“核心角色包”和“活跃角色包”

- 核心角色包：整季长期有效
- 活跃角色包：当前块真正会上场、会施压的人

### 4. 什么叫“逐集细纲真源”

- 详纲层输出的逐集细纲，是剧本当前集唯一正式依据
- 任何共享摘要、阶段引导、批次提示、测试口都必须从这里派生

### 5. 什么时候才能恢复放量测试

只有同时满足下面 5 条，才允许重新开测：

- 粗纲已按 `10 集块` 组织
- 人物已切成“核心 + 活跃”双层
- 详细大纲已不再固定四段
- 剧本已按 `5 集` 批次统一喂法
- 文档、任务卡、代码三处口径一致

## 这轮先做什么，后做什么

固定顺序：

1. 先收代码里的主链真相
2. 再收详纲页和剧本批次这些用户可见口径
3. 再把计划、规则、系统主权、任务卡写成同一套话
4. 最后只做结构验收，不立刻继续大档放量

## 当前已落下去的改动

- 粗纲已开始支持 `outlineBlocks`
- 人物已开始支持 `roleLayer` 和 `activeBlockNos`
- 详细大纲已改成按规划块生成逐集细纲
- 详纲页已开始改成按块和段展示，不再固定四段
- 剧本批次已改成固定按 `5 集` 推荐
- 编译已重新通过

## 这轮明确不做什么

- 不用硬 gate 代替工序统一
- 不用重 recovery 代替主链设计
- 不继续拿旧的 `20 集` 或 `60 集` 结果冒充新主链结果
- 不边改工序边宣布“质量已经稳定”

## 结构验收清单

- 粗纲是否真的有 `outlineBlocks`
- 人物是否真的能表达“核心 + 活跃”
- 详细大纲是否真的不再依赖固定四段
- 剧本是否真的按 `5 集` 批次组织
- 当前集是否只从逐集细纲真源取数
- 文档、任务卡、代码是否只剩一套口径

## 最后一句

这轮先解决“唯一流程”和“唯一文档”，再恢复测试。

因为测试只是放大镜，不是修根因的工具。
