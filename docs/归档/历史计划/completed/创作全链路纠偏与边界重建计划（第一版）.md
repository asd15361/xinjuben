# 创作全链路纠偏与边界重建计划（第一版）

更新时间：2026-03-20

## 2026-03-20 当前唯一执行口径

先暂停继续放量测试，先把“粗纲、人物、详细大纲、剧本”四层统一成一套唯一流程，再恢复分档测试。

当前只认这一套主链：

1. 正式底稿层
2. 整季总方向
3. `10 集` 粗纲块
4. `核心角色包 + 当前 10 集活跃角色包`
5. `10 集` 详纲块
6. `5 集` 剧本批次

四个关键定义：

- `10 集规划单位`：粗纲和人物活跃范围的默认块单位。
- `10 集详纲单位`：详细大纲跟着规划块走，不再固定四段。
- `5 集剧本单位`：剧本共享一份批次前情、压力线和活跃人物窗口。
- `逐集细纲真源`：剧本当前集任务只能来自详细大纲里的对应逐集细纲。

恢复测试前必须同时满足：

- 工序单位已经统一到代码和文档
- 剧本当前集只剩一个取数口
- runtime 已退回调度位，不再反向定义工序
- 任务卡、规则、系统主权、worklog 口径一致
- 结构验收通过

## 一句话结论

之前反复犯的错，不是某个 prompt 词写得硬。

真正的错是：

- 把整条链路拆开各修各的
- 哪层炸了就盯哪层补
- 今天说“只看剧本”
- 明天说“只看详纲”
- 后天又说“只是 parser 问题”

结果就是：

上游没守边界，下游被迫背债；  
计划越来越多，主路反而越来越不清楚。

这份计划只解决一件事：

把 `粗纲 -> 人物小传 -> 详细大纲 -> 剧本` 重新拉回一条链，按统一原则一起纠偏，不再各层各讲一套。

---

## 这次真正要修的本质问题

不是“哪层写不出来”。

是下面这 5 件事：

1. 没有始终按同一套工序边界做事
2. 没有始终按 A+B+C 写 prompt
3. 一出问题就滑回“加规则、加补丁、加解释、加兜底”
4. 把局部测试当全链路完成
5. 计划之间口径打架，今天刚定的原则，晚上又被新计划带偏

---

## 这次先纠什么错

### 错 1：把“阶段问题”误当“单点问题”

今天已经反复出现了：

- 剧本炸了，就只盯剧本
- 详纲跑不出，就只盯详纲
- 解析崩了，就只盯 parser

这就是老错。

因为下游写不出来，很多时候不是下游自己不会写，而是上游交下来的东西根本不是这一层该吃的输入。

### 错 2：把“优化”又做回“限制”

表面上一直在说“优化不限制”，  
但实际动作经常还是：

- 多写规则
- 多写职责说明
- 多写禁止示例
- 多加一层守卫
- 多补一段解释

这还是在加框子，不是在给梯子。

### 错 3：把“计划完成”误当“问题解决”

今天很多计划的写法，本质上还是：

- 计划拆得很细
- 阶段写得很满
- 但真正打没打到根因，不一定

大白话说：

像是在写作业，不是在修系统。

### 错 4：把“局部真测”误当“主链成立”

已经被今天的事实推翻过一次：

- 首场好，不等于 60 集好
- 剧本局部好，不等于整链好
- 详纲能出，不等于详纲这层定义是对的

以后不能再拿某一层、某一段、某一轮来冒充全链路。

### 错 5：计划本身也在越位

今天有些计划的问题，不是执行差。

而是计划自己就已经越位了：

- 详纲计划在教详纲怎么兼顾卖点、宣传语、阶段总述、逐集展开
- 剧本计划在默认剧本要替上游补格式债、补内容债、补结构债
- 总计划在讲全链路，但落到分计划后又重新碎掉

这说明不是只要“执行更认真”就行，  
而是计划层本身就需要收口。

---

## 当前全链路正确口径

后面所有计划和改动，只认这一套。

### 1. 粗纲层只负责什么

- 主线推进
- 每集发生什么
- 关键节点
- 关键人物清单
- 结局方向

粗纲层不负责：

- 补完整前史
- 提前写情绪渐变
- 提前写每场怎么演

### 2. 人物小传层只负责什么

- 前史
- 动机
- 关系
- 短板
- 触发条件
- 行动抓手

人物小传不负责：

- 重写粗纲主线
- 替详纲安排逐集冲突流程
- 替剧本写正文

### 3. 详细大纲层只负责什么

- 把粗纲逐集展开半步
- 把人物抓手挂进每集推进
- 补出更具体的动作、压强、代价、钩子

详细大纲不负责：

- 重写粗纲正式事实
- 重写人物正式动机
- 一边写阶段总表，一边写全量明细，还要自己做创作教学

### 4. 剧本层只负责什么

- 把前面已经确认好的东西写成可演场次
- 吃 A（前情）+ B（当前集任务）+ C（最小合同）

剧本层不负责：

- 替粗纲补主线
- 替人物补动机
- 替详纲补结构
- 一边创作，一边背一堆额外审计任务

---

## 当前链路里的真问题清单

### 粗纲层的问题

不是“不能产出”。

而是有风险继续回到两种歪路：

1. 把主线推进写成大段说明
2. 把同一轮推进重复抹平到很多集里

所以粗纲层现在要防的，不是“能不能出”，而是“是不是仍然只在平铺同一种压力”。

### 人物小传层的问题

不是“人物数量不够”。

而是人物小传很容易继续多做两件不该做的事：

1. 替详纲预排冲突流程
2. 替剧本提前写戏味

人物层要交的是发动机，不是半成品剧情。

### 详细大纲层的问题

这是今天最明显的一层。

真问题不是“禁止词太多”，而是：

1. 输入太杂
2. 任务太胖
3. 输出太重
4. 同时想写阶段总述和逐集细纲

所以它不是没词写，是任务定义就错了。

### 剧本层的问题

剧本层已经证明过两个真问题：

1. prompt 容易被补丁越堆越胖
2. 下游一旦开始替上游背债，格式和内容都会飘

今天虽然已经把剧本 prompt 朝 A+B 收回了一步，但还没形成“全链路都按同一标准收”的闭环。

### 计划层的问题

这是今天最容易被忽略、但其实最该改的一层。

今天计划层的真问题有 4 个：

1. 分计划各讲各的，没有一份总计划兜住全链路边界
2. 有的计划在修文风，不是在修工序定义
3. 有的计划默认“当前炸点就是根因”
4. 有的计划已经被事实推翻，但口径还没统一退场

---

## 这次总计划的唯一原则

以后所有子计划，只能从这 6 条往下长，不能再自己长一套。

1. 只做优化，不做限制
2. 提示词优先，硬编码靠后
3. 三道工序各守边界，谁也不替谁补根因
4. 正式事实层和 AI 推测层分开
5. A+B+C 只写本层需要的东西
6. 验收只认整链，不认局部冒充

---

## 执行总方案

### 第一阶段：先统一计划口径，不再计划互相打架

目标：

- 先把今天所有还在主导协作的计划，统一成一套口径

要做的事：

- 标出哪些计划还是有效的
- 标出哪些计划只能保留阶段成果，不能再拿来指挥当前主线
- 标出哪些计划已经被新事实推翻
- 补一份“当前唯一总计划”，让分计划以后都挂在它下面

完成标准：

- 后面再看文档时，不会同时看到两套相反说法

### 第二阶段：重新定义全链路四层职责

目标：

- 不再一层出问题就去另一层补

要做的事：

- 给粗纲、人设、详纲、剧本分别写清：
  - 只负责什么
  - 不负责什么
  - 输入是什么
  - 输出是什么
  - 不能越位到哪一层

完成标准：

- 再遇到问题时，能一眼找到第一次越界的是哪层

### 第三阶段：把四层 prompt 一起收回 A+B+C

目标：

- 不再只改单层 prompt

要做的事：

- 粗纲：只留正式底稿 + 当前层任务 + 最小输出合同
- 人物：只留主线需要的人物抓手 + 当前层任务 + 最小输出合同
- 详纲：只留粗纲逐集 + 人物抓手 + 逐集展开任务 + 最小输出合同
- 剧本：只留前情压缩 + 当前集任务 + 最小输出合同

完成标准：

- 四层 prompt 都不是说明书
- 四层 prompt 都只让模型做本层该做的事

### 第四阶段：把主链和补丁链彻底分开

目标：

- 不再让补丁偷偷回到主生成链

要做的事：

- 主链只保生成需要的最小输入
- audit / repair / fallback / postflight 全退到边上
- 不准再把这些补丁块重新塞回主 prompt

完成标准：

- 主链干净
- 补丁只在补丁位工作

### 第五阶段：把验收从“单层过”改成“整链过”

目标：

- 不再用局部样本自我安慰

要做的事：

- 重新定义四层联验口径：
  1. 粗纲是否真在推主线
  2. 人物是否真提供发动机
  3. 详纲是否真把逐集展开清楚
  4. 剧本是否真只是把上游结果写成戏
- 重新定义长度验收：
  - 10 集
  - 60 集
  - 更长长度只看“单棒负担有没有继续恒定”

完成标准：

- 以后谁再说“好了”，必须能拿整链证据

### 第六阶段：清今天留下的错误口径和错误惯性

目标：

- 不让今天的错明天继续复发

要做的事：

- 清掉还在主路上起作用的旧判断
- 清掉已经被推翻但还挂着“完成”的计划口径
- 明确写死：
  - 不能哪层炸了就只修哪层
  - 不能再用“加规则”冒充“优化”
  - 不能再用“阶段完成”冒充“问题解决”

完成标准：

- 明天再接着干的人，不会顺着旧错继续走

---

## 本轮先后顺序

这次不能再乱顺序。

固定这样做：

1. 先收计划口径
2. 再收四层职责
3. 再收四层 prompt
4. 再清主链/补丁链边界
5. 最后做整链真测

原因很简单：

如果前面三步不先收，  
后面改得再多，也只是继续边修边歪。

## 验收标准

这次只认下面 4 条：

1. 四层职责说得清，而且互不越位
2. 四层 prompt 都收回 A+B+C
3. 主链和补丁链彻底分开
4. 10 集、60 集都能按整链口径验，不再靠单层样本冒充

只要缺一条，就不算这次纠偏完成。

之前犯下的错，不是“某次改坏了”。

也不只是 parser，
