# 剧本格式验收标准（第一版）

## 定义权

- 剧本层唯一最高验收样本：`D:\我在八零年代当后妈》.docx`
- 当前仓库里旧的 `Action / Dialogue / Emotion` 只降级为内部摘要，不再算最终剧本
- 验收先看“是不是样本这种正式剧本”，再看内容好不好

## 本质问题

- 现在真正的问题不是“能不能写完”
- 真正的问题是：系统的生成、展示、修补、验收没有统一认同一个最终剧本真相
- 只要主链写的是正式剧本，但测试、导出、UI、修补还在按旧三段稿裁判，这个问题就没解决

## 正式剧本骨架

- 每集必须有 `第X集`
- 每集通常 `1-2` 场，最多 `3` 场
- 每个场次标题都必须写成 `场次编号 + 日/夜 + 内/外 + 地点`
- 每个场次都必须有 `人物：`
- 动作必须用 `△` 起头
- 对白必须写成 `角色名：内容`
- 括号情绪和动作只能夹在对白里，不能回到大段说明稿

## 节奏合同

- 每集目标字数：`450-750`
- 低于 `300` 视为空
- 高于 `900` 视为长稿风险
- 开头必须冷开场，先落动作，不先复述前情
- 集内信息必须高密度，优先外显冲突，不靠说明腔总结
- 集尾最后 `1-2` 行必须留下下一集可直接承接的动作、威胁、误会或揭露

## 每集最低门槛

- 必须出现 `第X集`
- 必须至少有 `1` 个合法场次标题
- 必须至少有 `1` 行 `人物：`
- 每个场次必须至少有 `1` 条 `△动作`
- 每个场次必须至少有 `2` 条对白
- 不能残留 `Action:` / `Dialogue:` / `Emotion:`

## 旧格式兼容原则

- 旧数据允许读取
- 旧数据允许被包成最小可显示剧本
- 但旧数据必须标记为“旧格式待重写”
- 旧格式永远不能作为新一轮剧本验收通过结果

## 验收顺序

1. 先验格式有没有对齐样本骨架
2. 再验节奏有没有接近样本密度
3. 最后才看内容质量和人物区分

## 当前实现要求

- 生成主链直接产出正式剧本正文
- UI 直接展示正式剧本正文
- 手动补写直接编辑正式剧本正文
- 搜索、筛选、导出、修补、审计统一优先读取 `screenplay`
- 测试默认做板块测试，不再只拿整链“跑完”冒充通过
