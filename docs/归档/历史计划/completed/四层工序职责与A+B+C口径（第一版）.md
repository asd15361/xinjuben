# 四层工序职责与A+B+C口径（第一版）

更新时间：2026-03-19

## 目的

这份文档只做一件事：

把 `粗纲 -> 人物小传 -> 详细大纲 -> 剧本` 四层到底各干什么，统一写死。

以后再出问题，先看第一次越界的是哪层，  
不要再哪层炸了就让别层补。

---

## 统一原则

四层都只按 A+B+C 组织。

### A 是什么

- 本层真正需要吃的前情和正式事实

### B 是什么

- 本层当前唯一任务

### C 是什么

- 本层最小输出合同

不允许把这些东西重新写胖成：

- 上层哲学讲义
- 一堆写法教学
- 一堆禁止示例
- 一堆补丁块

---

## 第一层：粗纲

### 这一层只负责什么

- 主线推进
- 每集主要事件
- 每集阻力
- 每集钩子
- 关键人物清单
- 关键节点
- 结局方向

### 这一层不负责什么

- 补完整前史
- 提前写情绪渐变
- 提前写场景正文

### A

- 正式创作底稿
- 用户明确给出的角色、关系、设定、需求

### B

- 把底稿整理成按集可推进的主线骨架

### C

- `storyIntent`
- `outline`
- `episodes`
- `facts`

---

## 第二层：人物小传

### 这一层只负责什么

- 前史
- 动机
- 关系
- 怕失去什么
- 会怎么施压
- 什么条件下会行动
- 最后怎么变

### 这一层不负责什么

- 重写粗纲主线
- 替详细大纲排逐集冲突
- 替剧本写戏

### A

- 正式创作底稿
- 当前粗纲主线
- 已明确的人物和关系

### B

- 把会推动主线的人物压成可执行的行动抓手

### C

- `characters`

---

## 第三层：详细大纲

### 这一层只负责什么

- 把粗纲逐集展开半步
- 把人物抓手挂进每一集
- 补出更具体的动作、压强、代价、钩子

### 这一层不负责什么

- 重写粗纲正式事实
- 重写人物动机
- 一边写阶段总表，一边写全量明细，还兼做创作教学

### A

- 粗纲逐集
- 人物行动抓手
- 当前集数和主冲突

### B

- 把每一集展开成可直接写剧本的逐集细纲

### C

- `episodes`

说明：

- 四段视图如果还要保留，只能走派生层
- 不再和逐集细纲绑成同一轮主生成

---

## 第四层：剧本

### 这一层只负责什么

- 把上游确认好的结果写成可演场次
- 吃前情、吃本集任务、按最小合同落正文

### 这一层不负责什么

- 替粗纲补主线
- 替人物补动机
- 替详纲补结构
- 一边创作一边背很多额外审计任务

### A

- 定长前情压缩
- 最近两场

### B

- 当前集任务
- 当前集核心冲突
- 当前集目标和变化

### C

- `Action`
- `Dialogue`
- `Emotion`

---

## 现在统一认定的越界信号

只要看到下面这些，就说明某层又开始越界了：

1. 这一层开始替下一层写答案
2. 这一层开始替上一层补根因
3. prompt 开始重新变成说明书
4. 主链里重新塞回补丁块
5. 输出合同同时要求两份主任务

---

## 最后一句

四层真正要守住的，不是“每层都写很多话”。

而是：

每层只做自己的事，  
每层只吃自己该吃的输入，  
每层只交自己该交的结果。
