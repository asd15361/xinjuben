# 优秀剧本稳定产出最终验收计划（第一版）

更新时间：2026-03-18 18:05

## 这份计划解决什么本质问题
- 这份计划不是再追“第几集会不会掉”。
- 现在真正的本质问题只有一句话：这条创作主路到底能不能稳定产出优秀的内容。
- 前面已经先把主喂法收成唯一公式：`A + B`。
- `A` 是固定提示骨架，`B` 是“前情提要 + 当前要写的大纲”。
- 既然每一轮都只能喂 `A + B`，那后面如果还会忽强忽弱，根因就不该再怪总长度，而该回头看两件事：
  - `A` 有没有把写法钉准
  - `B` 有没有筛得够准、够值、够轻
- 所以这份计划要解决的，不是“多跑几轮”，而是把三环都验清：这套主路到底能不能稳定、持续、可重复地交出优秀结果。

## 先钉死唯一验收标准
- 最终验收标准只有一句话：
- 这个项目必须可以稳定产出优秀的剧本内容。
- 这里的“稳定”不是偶尔一轮好看。
- 这里的“优秀”也不是文风热闹。
- 真正要同时成立的是 3 件事：
1. 三环都能稳定出来，不靠运气，不靠偷偷换更重喂法。
2. 结果质量真的好，不是只跑通、只成形、只看着像。
3. 主路还是正路，没有退回兜底、硬限制、补丁链、第二裁判口。

## 这份计划为什么是从本质出发
- 能被推翻的，不叫本质。
- “16 到 25 发虚”这种说法能被样本、时机、裁判口推翻，所以它不是本质。
- “60 集能跑完一次”也能被运气推翻，所以它也不是本质。
- 真正推不翻的是下面这条：
- 如果系统每一轮喂给模型的东西不是同一个结构，或者同一个结构里 `A` 和 `B` 质量不稳，那结果就不可能长期稳定优秀。
- 所以后面所有动作都只围着这一条做：
  - 不让主喂法再长胖
  - 不让同一个意思长出第二套说法
  - 不把下游问题甩给上游之外的工序兜底
  - 不把“能跑”误报成“优秀”

## 当前已经坐实的硬事实
- 主路和修补路都已经收回 `A + B`，不再允许越往后偷偷多塞上下文。
- `npm run typecheck` 和 `npm run build` 已通过，说明这轮主喂法收瘦后没有把工程带崩。
- 中段同压复打已经过了，说明现在主路不会因为往后几棒就自动变重。
- 当前内容层真正要盯的，不再是“会不会升不出来”，而是：
  - `B` 的前情提要筛得准不准
  - 模型有没有照着 `B` 写
  - `A` 对正文写法尤其对情绪段钉得够不够准
- 接下来正式进入最后重刺阶段，所有验收都要按三环同时记“质量 + 耗时”。

## 这份计划只看三环
1. 粗略大纲 + 人物小传
2. 详细大纲
3. 剧本生成

## 三环统一判断口径
- 每一环都必须同时回答 5 个问题：
1. 能不能稳定生成出来。
2. 内容质量到底怎么样。
3. 生成要多久。
4. 这一环的问题，根因到底落在哪一层。
5. 这次有没有真正推动“稳定产出优秀内容”这个本质问题。
- 以后不允许只报“通过了”。
- 必须拆开报：
  - 链路有没有通
  - 质量有没有过线
  - 时间是不是可接受

## 三环统一记录模板
- 每次正式记录都按同一模板写：
1. 样本口径：这次是 fresh 起跑、旧样本续跑，还是局部板块复测。
2. 测试范围：这次只测哪一环、哪一段。
3. 输入口径：本轮固定 `A + B`，有没有额外长出别的东西。
4. 结果状态：成功 / 失败 / 超时 / 页面异常。
5. 内容质量：优秀 / 可用但不够好 / 不过线。
6. 主要优点：最多写 3 条。
7. 主要问题：最多写 3 条。
8. 生成耗时：开始时间、结束时间、总秒数。
9. 根因归层：是 `A`、`B`、粗纲层、详纲层，还是剧本层本身。
10. 处理动作：继续下一环，还是退回上一层重做。

## 样本策略
- 这轮不再拿大而乱的整链一把梭做主验证。
- 正式验收分成两类样本：

### 第一类：主样本
- 用 `10` 集样本做三环完整验收。
- 原因不是 `10` 集最重要，而是它最适合把三环都看清。
- 这一类样本主要回答：
  - 粗纲和人物小传质量稳不稳
  - 详细大纲质量稳不稳
  - 剧本内容值不值
  - 三环耗时合不合理

### 第二类：补位样本
- 用已经站住的 `20` 级、`30` 级剧本位点做补位复看。
- 这一类样本不再重验“压力会不会越写越大”，因为这层已经先站住。
- 它只回答一件事：换了处境以后，`B` 还准不准，正文质量会不会漂。

## 执行阶段

### 第一阶段：先锁死唯一裁判口
- 把当前执行计划切到这份计划。
- 当前任务卡同步到这份计划上。
- 后面所有汇报、阶段记录、测试记录，都只认这份计划的口径。
- 完成标准：仓库里当前执行计划已经切过来，不再一边说长度无关，一边又按旧口径报结果。

### 第二阶段：先把三环记录板立起来
- 不急着再跑一堆测试，先把三环统一记录口径写死。
- 每一环都要有：质量结论、耗时、根因归层、下一步动作。
- 完成标准：后面任何一轮结果，都能直接塞进同一张记录板，不会再临时乱写。

### 第三阶段：正式验第一环
- 只看“粗略大纲 + 人物小传”。
- 先跑主样本。
- 每次只判断 3 件事：
  - 人物是不是立得住
  - 粗纲是不是能带动后面往下走
  - 时间是不是稳定可接受
- 这一环不允许拿后面的详纲或剧本来补功。
- 完成标准：第一环自己单独站住，能明确回答“这一步好不好、稳不稳、慢不慢”。

### 第四阶段：正式验第二环
- 只看“详细大纲”。
- 判断重点不是字数，也不是分了几段。
- 真正要看的是：它是不是把当前这一段的推进、关系、代价和下一步钩子讲清了，能不能直接喂给剧本层往下写。
- 这一环也必须单独记耗时。
- 完成标准：第二环自己单独站住，不靠剧本层帮它补结构债。

### 第五阶段：正式验第三环
- 只看“剧本生成”。
- 这一步分两种看法：
  - 主样本看三环完整链路下的剧本质量和耗时
  - 补位样本看 `20` 级、`30` 级位点在不同处境下稳不稳
- 这一环重点不是再证明“会不会越往后越重”。
- 这一层重点是看：
  - 模型有没有照着 `B` 写
  - 冲突、代价、钩子有没有落到戏里
  - 情绪段有没有又写回解释和复盘
  - 单场是不是太挤、太满、太爱总结
- 完成标准：第三环能明确给出“优秀 / 可用但不够好 / 不过线”的正式判断。

### 第六阶段：按归层原则回改
- 如果第一环不好，回第一环改，不准甩给后面。
- 如果第二环不好，回第二环改，不准让剧本层兜底。
- 如果第三环不好，优先按下面顺序回改：
1. 先看 `B` 的筛选是不是太满、太散、太不值。
2. 再看 `A` 对正文写法的约束是不是太松。
3. 最后才看剧本层有没有独立结构问题。
- 这一阶段的目标不是快，而是别再错层修。
- 完成标准：每个问题都能回到它第一次写错真相的那一层去改。

### 第七阶段：做总判断
- 三环结果收齐后，统一做一次总判断。
- 这次总判断必须同时回答 4 个问题：
1. 三环是不是都稳定。
2. 三环是不是都达到优秀内容标准。
3. 三环耗时是不是可接受。
4. 主路有没有滑回第二套喂法、第二裁判口或补丁链。
- 只有这 4 个问题都答“是”，这份计划才算完成。

## 每一环的过线标准

### 第一环过线标准
- 能稳定产出粗略大纲和人物小传。
- 人物不是空话，能看出谁在推戏。
- 粗纲不是流水账，能看出后面怎么接。
- 耗时没有明显失控。

### 第二环过线标准
- 详细大纲不是把粗纲重排一遍。
- 能看出这一段到底发生了什么变化。
- 能直接给剧本层当“当前要写的大纲”用。
- 耗时没有明显失控。

### 第三环过线标准
- 剧本不是只像剧本，而是真的有戏。
- 人物说话有区分，冲突不发虚，代价能认账，钩子能续命。
- 情绪不是解释出来的，而是场里打出来的。
- 模型大体照着 `B` 写，没有明显跑偏。
- 耗时没有明显失控。

## 三环失败时的唯一退回规则
- 第一环不过线：回粗纲和人物层，不动后面两环的兜底。
- 第二环不过线：回详细大纲层，不允许剧本层替它补结构。
- 第三环不过线：只许回 `A` 和 `B`，不准先长拦截、长恢复、长补丁链。
- 如果哪一环只是“跑得出，但质量差”，也按不过线处理。
- 如果哪一环只是“质量好，但耗时明显失控”，也不算真正过线。

## 这份计划明确禁止的假动作
- 因为总长度变长，就偷偷给当前一步加料。
- 第 1 轮一套提示词，第 20 轮再来一套更重上下文。
- 拿下游好结果，掩盖上游没站住。
- 拿“能跑通”冒充“内容优秀”。
- 一出问题就先加 gate、命中词、阻塞、重型 recovery、补名单。
- 同一个意思在多份文档、多块提示、多张卡里重复掌权。

## 这份计划完成后的状态
- 以后再看这个项目，不再先问“卡在第几集”。
- 而是先问：三环里哪一环没站住，`A` 和 `B` 到底哪一层还不够准。
- 项目也不再靠“某一次跑出来了”自我安慰。
- 而是能拿出成套记录证明：
  - 三环都稳
  - 三环都好
  - 三环时间可接受
  - 主路一直是正路
