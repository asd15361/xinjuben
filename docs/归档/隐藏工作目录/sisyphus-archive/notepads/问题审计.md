## T12可见结果与正式放行分离（第一切片）

**创建时间**：2026-03-23

**目的**：为 visibleResult 和 formalRelease 建立显式契约/类型层，使项目不再仅依赖隐式组合如 `generationNotice` + `failurePreview` + `generationPlan.ready`。

### 已创建文件

`src/shared/contracts/visible-release-state.ts`

### 核心类型

1. **VisibleResultStatus**：'none' | 'pending' | 'visible' | 'failed'
   - 表示用户可见的结果状态
   - 'failed' 状态允许用户查看和分析失败结果

2. **FormalReleaseStatus**：'blocked' | 'pending' | 'released'
   - 表示正式放行门状态
   - 'released' 需要独立条件检查，不能仅因为 visibleResult 存在就自动放行

3. **VisibleResultState**：包含状态、描述、payload、failureResolution、updatedAt
4. **FormalReleaseState**：包含状态、描述、blockedBy 列表、evaluatedAt
5. **GenerationResultState**：组合 visibleResult 和 formalRelease 的完整结果

### 关键语义不变性

- visibleResult 可以独立于 formalRelease 状态存在
- 即使 formalRelease 为 'blocked'，visibleResult 仍可为 'visible' 或 'failed'
- formalRelease 的 'released' 状态需要通过独立门条件检查

### 后续工作（T12 UI adoption）

1. 在 IPC handler 中集成 GenerationResultState 返回类型
2. 在 renderer 中消费 GenerationResultState 而非隐式组合
3. 更新 ProjectSnapshotDto 以包含正式的两状态字段
4. 在 useScriptGenerationRuntime 中连接新契约
5. 更新 ProjectGenerationBanner 使用新的可见结果契约
