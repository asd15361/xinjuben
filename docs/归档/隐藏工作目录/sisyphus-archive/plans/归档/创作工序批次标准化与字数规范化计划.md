# 创作工序批次标准化与字数规范化计划（第一版）

更新时间：2026-03-20

## 一句话结论

把粗纲、人物、详细大纲、剧本四层统一成一套合理的批次生成逻辑和字数标准。

---

## 问题诊断

### 当前实际问题

| 层级     | 问题                                       | 证据             |
| -------- | ------------------------------------------ | ---------------- |
| 粗纲     | 整季一次性生成，60集可能6000字输出，超窗口 | 每集100字估算    |
| 人物     | 全部一次性生成，没有按块区分核心/活跃      | 最多12个全量生成 |
| 详细大纲 | 按10集块，但字数无标准                     | prompt无限制     |
| 剧本     | 每集独立生成，字数标准只有剧本层有         | 500-800字/集     |

---

## 解决方案

### 各层批次策略和字数标准

#### 第一层：粗纲

| 项目       | 策略                                |
| ---------- | ----------------------------------- |
| 生成方式   | 按10集块分次生成（块1 → 块2 → ...） |
| 每块集数   | 10集/块                             |
| 字数上限   | 每集≤100字                          |
| 总输出估算 | 约1000字/块                         |

#### 第二层：人物

| 项目     | 策略                                    |
| -------- | --------------------------------------- |
| 生成方式 | 核心角色一次性全生成 + 活跃角色按块生成 |
| 核心角色 | 贯穿全季的角色，一次性全部生成          |
| 活跃角色 | 当前10集块需要的人物，按块生成          |
| 字数上限 | 每人不超过400字                         |
| 最大数量 | 核心≤8个，活跃每块≤6个                  |

#### 第三层：详细大纲

| 项目     | 策略                                  |
| -------- | ------------------------------------- |
| 生成方式 | 按10集块生成                          |
| 每块集数 | 10集/块                               |
| 输入     | 当前块的粗纲 + 该块的活跃角色         |
| 字数上限 | 每集≤300字                            |
| 内容要求 | 每集包含：场景+人物+张力目标+场尾钩子 |

#### 第四层：剧本

| 项目     | 策略                            |
| -------- | ------------------------------- |
| 生成方式 | 每集独立生成prompt，5集批次执行 |
| 批次执行 | 5集顺序生成                     |
| 字数标准 | 500-800字/集，字数上限850字     |

---

## 执行阶段

### 第一阶段：修改粗纲生成逻辑

**目标**：支持按10集块分次生成粗纲

**需要改的文件**：

- `src/main/application/workspace/generate-outline-and-characters.ts`
- `src/main/application/workspace/generation-stage-prompts.ts`

**验收标准**：

- [ ] 能按10集块生成粗纲
- [ ] 每集粗纲不超过100字
- [ ] 块和块之间的episodes编号连续

### 第二阶段：修改人物生成逻辑

**目标**：区分核心角色和活跃角色，按块生成

**需要改的文件**：

- `src/main/application/workspace/generate-outline-and-characters.ts`
- `src/main/application/workspace/generation-stage-prompts.ts`

**验收标准**：

- [ ] 核心角色一次性全生成（不超过8个）
- [ ] 活跃角色按10集块生成（每块不超过6个）
- [ ] 每人不超过400字

### 第三阶段：更新详细大纲生成逻辑

**目标**：确保按10集块生成，吃对应块的粗纲和活跃角色

**验收标准**：

- [ ] 每集细纲不超过300字，包含场景+人物+张力目标+场尾钩子

### 第四阶段：验证和测试

**测试顺序**：

1. 粗纲块生成（记录耗时、字数）
2. 人物生成（记录耗时、字数）
3. 详细大纲生成（记录耗时、字数）
4. 剧本生成（记录耗时、字数）
