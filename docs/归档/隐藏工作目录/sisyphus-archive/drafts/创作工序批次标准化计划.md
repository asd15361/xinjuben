# 计划：创作工序批次标准化与字数规范化

## 用户确认的方案

| 层级     | 生成方式                  | 字数上限   | 确认 |
| -------- | ------------------------- | ---------- | ---- |
| 粗纲     | 按10集块分次生成          | 每集≤100字 | ✅   |
| 人物     | 核心一次性 + 活跃按10集块 | 每≤400字   | ✅   |
| 详细大纲 | 按10集块生成              | 每集≤300字 | ✅   |
| 剧本     | 每集独立生成，5集批次执行 | 420-660字  | ✅   |

## 关键设计决策

1. **粗纲按10集块分次生成** — 因为60集×100字=6000字输出，超窗口
2. **人物分核心+活跃** — 核心贯穿全季一次生成，活跃按块生成
3. **详细大纲保持10集块** — 和粗纲/剧本批次对齐
4. **剧本每集独立生成** — 5集只是执行批次，不是喂入批次

## 批次依赖关系

```
底稿
  ↓
粗纲块1（10集）→ 人物核心（全季）+ 活跃角色块1
  ↓
粗纲块2（10集）→ 活跃角色块2
  ↓
...
  ↓
详纲块1（吃块1粗纲 + 块1活跃角色）
  ↓
详纲块2（吃块2粗纲 + 块2活跃角色）
  ↓
...
  ↓
剧本块1-批次1（5集，吃块1详纲）
  ↓
剧本块1-批次2（5集，吃块1详纲）
  ↓
剧本块2-批次1（5集，吃块2详纲）
  ↓
...
```

## 执行阶段

1. 修改粗纲生成逻辑（按10集块分次生成）
2. 修改人物生成逻辑（核心一次性+活跃按块）
3. 设定各层字数标准
4. 按板块测试验证
