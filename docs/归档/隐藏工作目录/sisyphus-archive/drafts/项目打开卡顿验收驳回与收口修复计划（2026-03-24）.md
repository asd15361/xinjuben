# Draft: 项目打开卡顿验收驳回与收口修复计划

## Requirements (confirmed)

- 用户要求：把刚才的分析做成计划。
- 当前目标：把“已做完”的错误判断改成正式的验收驳回后修复计划。
- 计划对象：项目打开卡顿、切页收口、stale data、弱缓存指纹、render 热路径、lint 与定向测试门禁。
- 验收基线：不能以“看起来顺一点”算完成，必须以仓内命令、grep、定向测试、静态门禁和用户真人手测关注点收口。

## Technical Decisions

- 计划类型：架构级 repair-and-hardening，不是新功能开发。
- 执行顺序固定为：权威收口 → stale data 清除 → render 热路径拆除 → 静态门禁 → 性能复验。
- Stage 切换唯一正式入口：renderer 不再允许直接调用 `window.api.workspace.changeProjectStage()`；统一走 `stage-session-service`。
- Project open 唯一正式入口：`open-project-session` 不能继续走 `getProjectSnapshot(projectId, undefined)` 这种全量快照旁路。
- `activeCharacterBlocks` 必须恢复单一正式生产者；CharacterStage 不能只删 effect 而不补 authoritative producer。
- `projectName` / `applyStageTransition` 只能有一个 owner；`useHomePageActions` 与 `stage-session-service` 不能双写。
- `script-plan-service` 的 fingerprint 优先保证正确失效，不优先追求缓存命中率。
- Script quality audit 必须从列表 render 主链退出；列表只保留轻量摘要与显式展开正文。
- lint 红线必须通过规则修正（忽略临时目录或等价正式处理），不能靠人工解释豁免。

## Research Findings

- `workspace-project-handlers.ts` 重复注册：`workspace:open-project-session` 两次，`workspace:switch-stage-session` 两次。
- `workspace-project-handlers.ts` 中 `open-project-session` 仍调用 `getProjectSnapshot(projectId, undefined)`，保留全量快照热路径。
- `stage-session-service.ts` 已存在，但 renderer 仍有多处直接 `changeProjectStage()` 调用，切页未真正收口。
- `stage-session-service.ts` 与 `useHomePageActions.ts` 都在写 `projectId/projectName/applyStageTransition`，且 `projectName` 被错误写成 `projectId`。
- `useStageStore.ts` 在 script hydration 与 character save 路径仍消费/保存 `activeCharacterBlocks`，但 CharacterStage 现状未提供新的 authoritative producer。
- `script-plan-service.ts` 当前 revision 只看 mode / targetEpisodes / 各长度，遗漏 `activeCharacterBlocks`、`detailedOutlineBlocks` 等关键输入语义。
- `useScriptSceneIndex.ts` 仍保留质量检查逻辑；`ScriptSceneList.tsx` 的 SceneCard 仍执行 `useScriptSceneQuality()` 和 `buildScreenplayFromStructuredScene()`。
- `npm run typecheck` / `npm run build` 可过，但 `npm run lint` 被 `.codex/tmp-hook-check/*` 等临时目录拖红；计划要求的 5 个 file-scoped tests 均不存在。

## Open Questions

- 无阻断用户问题。当前 repo 证据已足够直接生成修复计划。

## Scope Boundaries

- INCLUDE: IPC handler 唯一化、项目打开快照收口、stage session 全面接管切页、activeCharacterBlocks 正式生产者、home/session 双写清理、script plan 指纹重做、script quality audit 退出 render 主链、lint scope truthful 化、定向测试补齐、性能复验。
- EXCLUDE: 不相关 UI 重设计、不相关 IPC 框架重写、全局缓存系统重做、E2E/Playwright/浏览器自动化验收、长链路 smoke、用户真人手测替代。
