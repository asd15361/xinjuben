# 测试（目录说明）

这个目录只做一件事：
**作为测试脚本、测试说明、测试清单的唯一新入口。**

## 现在的口径
- 从现在开始，新建测试脚本、新建测试说明、新建测试清单，统一放这里。
- 历史 `tools/e2e` 是旧测试区，里面已经堆了大量脚本和测试产物；它还活着，但不再继续往里面加新的测试文件。
- 旧区迁移要单独做仓库清理任务，不和主线修 bug、fresh run 排查、根因修复混在一轮里做。

## 建议结构
- `测试/e2e/`：自动化测试脚本
- `测试/说明/`：测试说明和执行口径
- `测试/清单/`：分板块验收清单

## 当前状态
- 这个目录先作为唯一新入口和导航点。
- 历史脚本完全迁过来之前，涉及旧区脚本时，要在任务卡或 worklog 里写清楚“这是旧区脚本，不是新规则例外”。
- 测试产物继续放忽略目录，不把截图、用户数据目录和日志堆成仓库主内容。
