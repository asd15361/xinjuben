# E2E旧脚本逐支判无效台账（2026-03-22）

## 结论

- 本台账不是“旧脚本修复清单”，而是“旧脚本逐支判无效清单”。
- 判定标准不是“偶尔能不能跑绿”，而是“能不能作为正式产品证据”。
- 只要脚本存在以下任一问题，就不得进入正式验收名单：
  - 绕过真实 UI 主链
  - 直接改 `projects.json` 或制造测试状态
  - 默认复用历史 `userdata-*`
  - 使用旧字段、旧术语、旧状态格式做裁判
  - 吞错、瞎等、把停滞当完成
  - 验收合同与当前正式质量合同不一致

## 逐支判定

| 文件 | 原用途 | 当前判定 | 不得过审的核心原因 |
| --- | --- | --- | --- |
| `electron_real_project_gate_check.mjs` | 真实项目 gate 验收 | 删除 | 按钮不可点时直接 `page.evaluate()` 调内部生成 API，已不是 UI 验收 |
| `electron_real_golden_chain.mjs` | 真实全链路黄金链 | 删除 | 旁路内部 API、依赖旧入口、存在旧链路假通过风险 |
| `electron_real_script_60_check.mjs` | 60 集首批剧本检查 | 删除 | 旁路内部 API，且仍依赖旧详纲/旧链路判断 |
| `electron_real_script_check.mjs` | 单批剧本格式检查 | 删除 | 自带第二裁判口，合同写成 `300-900字 / 1-3场`，与正式合同不一致 |
| `electron_real_script_60_full_export.mjs` | 60 集长测导出 | 删除 | 旧入口、旧字段、observer 依赖、导出摘要继续带旧口径 |
| `electron_script_status_recheck.mjs` | script 状态复核 | 删除 | 直接裁剪 `scriptDraft`、清空 runtime 字段后回写 `projects.json`，测试在造状态 |
| `phase4-isolated-script-check.mjs` | 隔离样本验收 | 删除 | 注入样本到 `projects.json` 后再跑 app，测的是脚本塞入的数据，不是真实链路 |
| `electron_real_100ep_full_check.mjs` | 100 集长测汇总 | 删除 | 把 `generationStatus` 当旧字符串；PASS 只看数量，不看正式质量合同 |
| `electron_real_100ep_baton_check.mjs` | 100 集 baton 长测 | 删除 | 旧 `generationStatus` 判断、旧字段 `detailedOutlineSegments`、停滞/完成混判 |
| `baton-distribution-loop.mjs` | 批量 baton 观察 | 删除 | 30 秒无增量直接判 `batchDone=true`，把停滞当完成 |
| `script-batch-observer.mjs` | 统一观察器 | 删除 | `catch {}` / `.catch(() => null)` 吞错，completion 判断存在竞态，不能当裁判 |
| `electron_golden_chain.mjs` | 旧黄金链 | 删除 | 旧术语、乱码选择器、旧页面信号，先天不稳定 |
| `electron_failure_resume.mjs` | 恢复链验收 | 删除 | 默认吃二次加工的 `userdata-*`，且并未验证真实恢复动作本身 |
| `electron_targeted_script_test.mjs` | 定向脚本验证 | 删除 | 默认绑定历史 `userdata-*`，并带有裁剪/覆写项目状态路径 |
| `electron_workspace_stage_smoke.mjs` | workspace 阶段冒烟 | 删除 | 绑定历史 `userdata-*`，且使用旧按钮/旧页面口径 |
| `electron_seeded_real_regression.mjs` | seeded 真实回归 | 删除 | 直接改 `projects.json` 清 runtime 状态，属于 observer effect |
| `electron_seeded_resume_regression.mjs` | seeded 恢复回归 | 删除 | 依赖不稳定 seed 血缘，结果受上游污染脚本影响 |
| `electron_real_detailed_outline_check.mjs` | 详纲真实检查 | 删除 | 依赖旧 helper 与旧测试架构，不能作为正式验收入口 |
| `electron_real_outline_3run_check.mjs` | 三轮粗纲检查 | 删除 | 属于旧 real runner 体系，缺少当前正式合同和新链路准入保障 |
| `electron_real_outline_character_check.mjs` | 粗纲+人物检查 | 删除 | 属于旧 real runner 体系，只能给观察结论，不能判产品真相 |
| `golden_chain_smoke.py` | 旧黄金链烟测 | 删除 | 继续使用旧页面文案和旧术语，不再对应当前 UI |
| `ui_snapshot.py` | UI 标题快照 | 删除 | 快照对象仍是旧标题旧口径，留着只会制造噪音 |
| `ui-signals.mjs` | UI 信号常量 | 删除 | 模糊正则多匹配、旧术语残留，无法作为稳定按钮识别层 |
| `step6-verify.mjs` | 第 6 步验证 | 删除 | 绑定旧链路和历史输出目录，已不具备当前主链意义 |
| `detailed-outline-helpers.mjs` | 详纲测试 helper | 删除 | 服务对象是整批已退役脚本，不再保留旧 helper 公共口 |
| `mock-boundary.mjs` | real/mock 边界守卫 | 删除 | 守护对象已退役；单独保留只会让旧 runner 看起来还可用 |
| `root_cause_guards.mjs` | E2E 根因守卫 | 删除 | 自身存在 bare catch 吞错，且守卫对象多为已退役脚本 |
| `electron_real_60_benchmark_matrix.mjs` | 长测矩阵编排器 | 删除 | 只是在编排已失效脚本，不能独立成立 |

## 保留但不准冒充正式验收的工具

| 文件 | 用途 | 当前状态 |

| `e2e-output.mjs` | 输出目录清理 | 保留，但只是基础设施 |
| `electron_launch_smoke.mjs` | Electron 启动烟测 | 保留，但只验证壳启动 |
| `run_dev_mock.ps1` | 本地开发辅助 | 保留，但不是测试证据 |

## 最终判定

- 上表删除项全部“不允许过审”。
- 不是“先删，之后再看”，而是“已经完成逐支判无效”。
- 后续若要恢复 E2E，只能按新准入标准从零重建，不允许从这些脚本上修补转正。
