# 工序单位与合同统一标准表（2026-03-20）

更新时间：2026-03-20

## 目的

这张表只做一件事：

把当前主链里 `粗纲 / 人物 / 详细大纲 / 剧本` 四层的：

- 工序单位
- 生成批次
- 输入合同
- 输出合同
- 验收合同

一次写成唯一口径。

当前项目只允许：

- `10集规划`
- `10集详纲`
- `5集写作`
- `逐集细纲真源`
- `核心角色包 + 当前块活跃角色包`

这份表是当前执行层的标准表，
但它不是第二份计划文档；当前唯一计划入口仍然只有：

- `docs/plans/计划总表.md`

## 逐层展开

### 1. 粗纲层

#### 工序单位

- `10集规划单位`

#### 真实生成批次

- 当前仍是 `5集/次` 模型调用
- 但输出后必须拼回 `10集规划块`

#### 输入合同（概念）

- 正式创作底稿
- 用户明确给出的角色、关系、设定、需求
- 当前 story intent
- 已确认正式事实（如果存在）

#### 输出合同（概念）

- `storyIntent`
- `outline`
- `outlineBlocks`
- `summaryEpisodes`
- `facts`

#### 验收合同（概念）

- 规划块必须按 `10集` 组织
- 单集只交主线骨架，不提前越级写人物小传或剧本正文
- 正式事实链必须能继续流向详纲层

#### 实现落点（代码）

- 规划单位与块切分：`src/shared/domain/workflow/planning-blocks.ts` — `DEFAULT_OUTLINE_BLOCK_EPISODES = 10`、`buildOutlineBlocks()`。
- 粗纲生成 prompt：`src/main/application/workspace/generation-stage-prompts.ts` — 粗纲分块 prompt（已说明“10集规划块，内部按5集一批调用模型”）。
- 输入 gate：`src/main/application/input-contract/validate-stage-input.ts` — `targetStage: 'outline'` 分支校验 storyIntent/outline/facts 是否齐备。
- 输出实体：`src/shared/contracts/workflow.ts` — `OutlineDraftDto` / `OutlineBlockDto` / `OutlineEpisodeDto` / `FormalFactDto`。

### 2. 人物层

#### 工序单位

- `核心角色包 + 当前块活跃角色包`

#### 真实生成批次

- 当前代码口径：主运行链仍偏“整包生成人物，再切活跃层”
- 这件事当前要如实写明，不能误报成“已经完全按双层分轮运行”

#### 输入合同（概念）

- 正式创作底稿
- 当前粗纲主线
- 已明确人物关系
- 当前块主冲突

#### 输出合同（概念）

- `characterDrafts`
- `activeCharacterBlocks`

#### 验收合同（概念）

- 核心角色包必须独立存在
- 当前块活跃角色包必须能覆盖本块实际推进人物
- 不允许只剩整包角色、不认活跃层

#### 实现落点（代码）

- 角色块切分：`src/shared/domain/workflow/planning-blocks.ts` — `buildCharacterBlocks()`，按规划块 + activeBlockNos + 出现频次筛“核心+活跃”人物。
- 初次生成：`src/main/application/workspace/generate-outline-and-characters.ts` — 生成 `characterDrafts + activeCharacterBlocks` 并写入项目快照。
- 存储与注入：`src/main/infrastructure/storage/project-store.ts`、`src/renderer/src/store/useStageStore.ts` — 加载/持久化 `activeCharacterBlocks`。
- 剧本合同 gate：`src/shared/domain/script-generation/contract-policy.ts` — `buildScriptGenerationContract()` 要求存在 `activeCharacterBlocks`。
- 剧本 prompt 注入：`src/main/application/script-generation/prompt/create-script-generation-prompt.ts` — `findCharacterBlock()` + `activeCharacterNames` 进入 `buildCurrentEpisodeTask()`。

### 3. 详细大纲层

#### 工序单位

- `10集详纲块`

#### 真实生成批次

- `10集/块`
- 块内每集必须落 `summary + sceneByScene`
- `sceneByScene` 当前正式口径：每集 `2-3场`

#### 输入合同（概念）

- 当前粗纲块
- 人物行动抓手
- 已确认正式事实
- 当前块主冲突

#### 输出合同（概念）

- `detailedOutlineBlocks`（正式真源）
- `episodeBeats.summary`
- `episodeBeats.sceneByScene`
- `detailedOutlineSegments`（仅兼容投影，不是正式真源）

#### 验收合同（概念）

- 只认 `detailedOutlineBlocks + episodeBeats + sceneByScene`
- confirmed facts 必须真正进入详纲文本 / 逐集细纲
- `segments` 不能再主导门禁、ready 判断和正式裁判口

#### 实现落点（代码）

- 详纲生成 prompt：`src/main/application/workspace/generation-stage-prompts.ts` — 详纲 prompt JSON schema（summary + sceneByScene）、2-3场、500-800字要求。
- 详纲生成执行：`src/main/application/workspace/generate-detailed-outline.ts` — 写入 `detailedOutlineBlocks` 与兼容的 `detailedOutlineSegments`。
- 输入 gate：`src/main/application/input-contract/validate-stage-input.ts` — `targetStage: 'detailed_outline'` 分支校验 outline/characters/facts 及 blocks 是否齐备。
- 剧本前置 gate：同文件 `targetStage: 'script'` 分支，用 `detailedOutlineBlocks.sections[].episodeBeats[].sceneByScene` 做正式事实落点与结构检查。
- DTO 真源定义：`src/shared/contracts/workflow.ts` — `DetailedOutlineBlockDto` / `DetailedOutlineSectionDto` / `DetailedOutlineEpisodeBeatDto`。

### 4. 剧本层

#### 工序单位

- `5集批次上下文 + 单集逐集落地`

#### 真实生成批次

- 当前不是“5集一起出正文”
- 当前真实是：`5集/批` 共享上下文，但每集单独调模型写正文

#### 输入合同（概念）

- A：前情提要
  - 全历史粗压缩
  - 最近两场细摘要
- B：当前集任务
  - 当前 `5集` 批次前情
  - 当前集 `sceneByScene`
  - 当前块活跃人物
  - 当前集集尾压力目标

#### 输出合同（概念）

- 正式剧本正文
- `scriptBatchContext`
- 剧本目录 / 剧本摘要

#### 验收合同（概念）

- 每集 `2-3场`
- 每集 `500-800字`
- 必须是正式剧本正文格式，不是旧三段稿替身
- `repeatedOpeningEpisodes = []`
- 各集 `problems = []`

#### 实现落点（代码）

- 剧本输入合同 gate：`src/main/application/input-contract/validate-stage-input.ts` — `targetStage: 'script'`，只看 `detailedOutlineBlocks + sceneByScene + characters + confirmed facts`，不给 segments 第二真相口。
- 剧本合同 gate：`src/shared/domain/script-generation/contract-policy.ts` — `buildScriptGenerationContract()`，ready 只认 blocks + sceneByScene；activeCharacterBlocks 为必要条件之一。
- 剧本输出合同：`src/main/application/script-generation/prompt/create-script-generation-prompt.ts` — `buildOutputContract()` 写死 2-3 场、500-800 字、正式剧本格式要求。
- 剧本质量门：`测试/剧本/validate-screenplay-quality.mjs`、`测试/剧本/validate-screenplay-output.mjs` — 2-3 场、500-800 字、标题/人物/动作/对白/钩子等硬要求。
- 全档 E2E 验收：`tools/e2e/electron_real_script_60_full_export.mjs` + 其他 real\_\* e2e — 检查 `passedEpisodes`、`averageCharCount`、`repeatedOpeningEpisodes`、`problems` 等指标。

## 当前唯一判断

### 已经写死的事实

1. `10集` 是规划单位，不等于粗纲真实模型调用批次
2. `10集` 是详纲块单位
3. `5集` 是剧本批次上下文单位，不等于“5集一起写正文”
4. 剧本正文仍是逐集生成
5. 人物当前正式口径只认“核心角色包 + 当前块活跃角色包”
6. 剧本当前集只认“逐集细纲真源”

### 当前还没允许随意改口的点

1. 粗纲是否从 `5集/次` 改成 `10集/次`
2. 人物是否改成“核心先生成，再按块生成活跃角色”
3. 无 `sceneByScene` 时，到底直接阻断，还是保留兼容写作模式

这些都必须先在 `计划总表.md` 里改顺序、改口径，
不能在执行时边做边改定义。
