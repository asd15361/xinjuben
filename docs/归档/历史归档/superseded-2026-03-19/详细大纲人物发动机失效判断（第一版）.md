# 详细大纲人物发动机失效判断（第一版）

## 这份文档只回答一个问题
- 为什么人物小传明明已经进了详细大纲输入，最后却没有真正接管详细大纲。

## 先说结论
- 现在不是没有人物发动机。
- 现在是：人物发动机虽然进去了，但只是“被看见”，没有“被强制使用”。

所以最后产出的详细大纲还是事件在往前排，人只是跟着事件出现。  
这就是为什么它看起来有人物，实际上还是假详纲。

## 现在到底坏在哪

### 1. 人物信息只是挂在输入后面，没有变成阶段硬任务
- 现在详细大纲输入里，确实带了人物小传。
- 里面也写了：
- 表面样子
- 暗里压力
- 最怕失去什么
- 最想守住什么
- 什么时候会出手

但问题是：
- 这些信息只是被附在后面
- 没有被要求“必须落到四段里”

结果就是：
- 写出来也行
- 不写出来也能交卷

只要还存在这种情况，人物发动机就没有真正接进来。

## 2. 现行详纲还是按事件窗口在排，不是按人物动作在排
- 现在四段的写法，本质上还是：
- 这一段有哪些事
- 把这些事按顺序塞进去

而不是：
- 这一段是谁最先动
- 谁最先压
- 谁的软肋先被碰
- 哪个人因为怕失去什么而改打法

这说明当前推动详纲的主轴还是“事件窗口”，不是“人物发动机”。

## 3. 人物各自的作用没有被分配到四段
- 以这次真实样本为例：
- `少年守钥人` 的核心应该是“守什么、什么时候亮底”
- `反派恶霸` 的核心应该是“怎么加码施压”
- `小镇少女` 的核心应该是“什么时候把关系代价压上来”
- `山中妖物` 的核心应该是“什么时候把外压升级成灾难”

但现行详纲里，这些人并没有在四段里各自承担固定职责。
- 他们只是跟着每集事件顺手出现。

这说明详纲没有把人物从“名单”变成“发动机”。

## 4. 这为什么是本质问题
- 因为只要人物发动机不接进来，详纲就永远会往“事件流水账”退。
- 就算词写得再漂亮，也还是假详纲。
- 只要还能被一句“这些人只是出现在段里，不是真正在推段落”推翻，就说明本质问题没解决。

## 当前正式判断
- 现行详细大纲里，人物问题不是“人物不够多”。
- 不是“人物介绍不够长”。
- 不是“名字没写进去”。

**本质问题是：人物发动机现在还没有被强制接入阶段推进，所以详细大纲仍由事件在带，人没有真正带阶段。**

## 当前验收口径一句话版
- 只要新的详细大纲里，还看不出“这一段是谁在主推、谁在主压、谁在主反咬、谁把代价抬起来”，就一律算人物发动机没接进去。
