# 剧本正文质量升级与外部流程吸收计划（第一版）

更新时间：2026-03-19

## 2026-03-19 更正（先看这段）
- 最新 60 集真测已经推翻“这份计划可以代表 60 集稳定完成”的口径。
- 这份计划保留下来的，只是“局部正文质量有提升、首场和局部样本更像戏”的阶段成果。
- 当前真正主线已经切到：`docs/plans/60集剧本稳定生成专项修复计划（第一版）.md`
- 后续凡是要判断“到底有没有写出来”，都以 60 集 full export 真测为准，不再以这份计划的阶段结果代替。

## 这份计划解决什么本质问题
- 现在真正没封顶的，不是架构，不是长度无关，不是前端主路，也不是唯一真相源。
- 当前真正卡住的，是：系统已经能稳定产出粗纲、人物和详细大纲，但剧本正文离“稳定优秀”还差最后一截。
- 这一截不是“不会写”，而是“正文还不够像戏”，偶尔还会冒出内部工作标记、说明感偏重、对白偏长、光影和动作力度不够的问题。
- 所以这份计划要解决的，不是再动大架构，而是把剧本正文这最后一截真正压到“稳定优秀”。

## 当前事实（先钉死，不再空谈）
- 已完成并成立：
  - 根因 1-6 已收口
  - 长度无关 baton 架构已验证成立
  - 前端交互收口已完成
  - 唯一真相源最终封口已完成
- 当前质量真实状态：
  - 粗纲：好
  - 人物：好
  - 详细大纲：好
  - 剧本正文：部分合格，但还没到“稳定优秀”
- 当前正文已坐实的短板：
  1. 偶发内部工作标记泄露到最终产出，比如 `**`
  2. 情绪段有时还是说明文，不够可演
  3. 部分对白偏长，节奏不够短、不够准
  4. 整体已经够“讲清楚”，但离“有声音感、有光影感、有动作力度”还差一截
- 所以当前问题不在架构层，在剧本提示词主路和正文执行口。

## 这轮不做什么
- 不再重做长度无关架构。
- 不再回头大修唯一真相源。
- 不把这轮问题重新改成前端交互问题。
- 不把外部项目当“剧本能力老师”来学。
- 不现在开云端主权大工程。
- 不允许为了追质量，又长出一层 gate、命中词、重 recovery 或补丁链。

## 外部样本在这轮里只扮演什么角色
- 外部项目只当“产品流程和工程收口样本”来学。
- 不拿它们当“剧本能力样本”来学。
- 这轮学习只做一件事：看它们怎么把流程、状态反馈、板块衔接、工程收口做稳，再挑能直接服务我们当前正文质量升级的东西吸收回来。
- 学习是补强，不是主角；主角仍然是“把剧本正文压到稳定优秀”。

## 这轮最终验收标准
- 必须同时成立：
1. 粗纲、人物、详细大纲不能回退。
2. 剧本正文不再泄露内部工作标记。
3. 情绪段更像可演场次，不再主要靠说明补。
4. 对白更短、更准、更有逼压感。
5. 首场到后续场次都更像戏，不只是更像说明稿。
6. 外部样本学习已经转成我们自己的具体动作，而不是停在“看过了”。

## 执行阶段

### 第一阶段：冻结当前真相，不让后面再跑偏
- 目标：
  - 把“现在已经成立什么、现在真正差什么”写死。
- 动作：
  - 把当前真实结论固定成唯一口径：
    - 架构稳
    - 粗纲/人物/详纲好
    - 正文差最后一截
  - 把正文短板固定成 4 条：
    - 标记泄露
    - 说明感偏重
    - 对白偏长
    - 表演感和光影力度不足
- 完成标准：
  - 后续不再有人把这轮问题重新说成“架构没好”或“长度无关失败”。

### 第二阶段：先压掉最硬的低级问题
- 目标：
  - 先把不该出现的低级泄露和内部口气清干净。
- 动作：
  - 清掉正文里的内部工作标记、幕后提示痕迹、系统脑内话。
  - 情绪段禁止再靠解释句、括号补充、幕后说明来兜气氛。
  - 对白先做第一轮收短，先压掉最长、最拖节奏的口子。
- 完成标准：
  - 再跑样本时，正文里看不到内部标记和明显后台话术。

### 第三阶段：把正文从“讲清楚”压到“能演出来”
- 目标：
  - 让正文更像戏，不只是更像说明稿。
- 动作：
  - 强化动作力度：让冲突更落在身体和下一秒动作上。
  - 强化对白逼压：少讲概念，多讲当下要命的那一下。
  - 强化光影和声音感：让读出来就有画面和空气，不靠解释。
  - 强化情绪兑现：情绪更多通过动作、停顿、反应来出，不靠作者说明。
- 完成标准：
  - 同一场戏里，读者不用看解释，也能知道谁在逼谁、谁在硬撑、下一秒会发生什么。

### 第四阶段：做 3 轮同口径真测
- 目标：
  - 验证这不是偶尔一把写好，而是开始稳定写好。
- 动作：
  - 按同一条真实链路连续做 3 轮样本。
  - 每轮都按同一套口径看：
    - 是否还有标记泄露
    - 是否还有明显说明感
    - 对白是否还偏长
    - 首场是否更有动作、声音、光影和逼压
- 完成标准：
  - 不是只赢 1 次，而是 3 轮都过线。

### 第五阶段：边做边吸收外部样本，但只吸收流程和收口
- 目标：
  - 不闭门修自己，也不被外部项目带偏。
- 动作：
  - `waoo`：看产品壳子、用户入口、状态反馈和守卫落地
  - `魔音漫创`：看流程板块怎么串、可选步骤和必经步骤怎么收口
  - `openoii`：看多 agent 分工后，流程和状态怎么不打架
  - `huobao drama`：只扫长期运行和迁移经验，不单开主线
  - 每看完一个样本，只回答 3 个问题：
    - 最值得学的 3 条流程/收口动作是什么
    - 最不该学的 3 条是什么
    - 哪 1-2 条能立刻服务我们当前正文质量升级
- 完成标准：
  - 外部学习已经变成我们自己的具体动作，不再只是“看过”。

#### 第五阶段已写死的 `waoo` 学习结论
- 结论 1：学它的一条主路，不学它的热闹外壳。
  - 用户眼里只该看到“我现在在哪一步、下一步该去哪里”，不该同时看到一堆内部工序词、状态词和系统脑内话。
  - 我们要学的是“主路更直”，不是“页面更花、功能更多”。
- 结论 2：学它的双层状态反馈，不学假进度。
  - 用户既要知道“整个项目现在跑到哪”，也要知道“当前这一批/这一集/这一场正在做什么”。
  - 状态反馈要报位置，不要报热闹；要能看见继续点，不要再用假百分比和空倒计时糊过去。
- 结论 3：学它的静默守卫，不学重 gate 味。
  - 守卫只负责防止主路跑歪，并告诉用户该回哪一页补，不负责替用户写内容，也不负责另起一套解释口。
  - 所有守卫都只能挂在现有主链上，不能再长第二裁判口、第二发布口、第二解释口。

#### 第五阶段明确不学什么
- 不学“功能越多越高级”的外壳堆砌。
- 不学“前台摆很多规则让用户自己消化”的重 gate 做法。
- 不学“为了可解释，又新长一套创作说明系统”的第二写口。

### 第六阶段：把外部学习真正并回当前项目
- 目标：
  - 只吸收对当前正文质量升级真正有用的东西。
- 动作：
  - 只挑低风险、高价值的流程和收口动作并回当前项目。
  - 优先并回这几类：
    - 用户更容易看懂“现在生成到哪”
    - 用户更容易看懂“这场戏为什么这样写”
    - 工程上更不容易再长回第二口子
  - 不允许因为学习外部项目，就把当前主线变成重工程、轻创作。
- 完成标准：
  - 学习结果已经落到当前项目里，且没有把主线带偏。

#### 第六阶段详细执行版（照着做，不要自由发挥）

##### 第六阶段先后顺序
1. 先做“状态反馈并回”
2. 再做“场景解释并回”
3. 最后做“静默守卫并回”
- 原因：
  - 先让用户看懂自己在哪，再让用户看懂这一场为什么这样写，最后再把这些口径锁进唯一主链。
  - 不准倒着做。先上守卫、后补说明，最容易又长出第二口子。

##### 第六阶段-A：状态反馈并回
- 要解决的本质问题：
  - 用户现在虽然能走主路，但还不够一眼看懂“整个项目在哪一步”和“这一小步正在干嘛”。
- 这一步只做什么：
  - 把状态反馈统一成双层：
    - 第一层：整个项目当前处于哪一阶段
    - 第二层：当前这一批 / 这一集 / 这一场正在执行什么
  - 把“生成中 / 已完成 / 失败 / 待补”这些状态统一成人话，不准夹带内部工序词。
  - 把“继续去哪”做成明确下一步，不准让用户自己猜。
- 这一步绝对不做什么：
  - 不准重上假百分比、假秒数、空转进度条。
  - 不准把后台内部字段原样甩给用户。
  - 不准为了显示更细，就新长一套前台状态机。
- 执行动作：
  - 盘一遍当前主路上已有的阶段状态、生成状态、批次状态。
  - 只保留一套前台展示口径，删除重复说法。
  - 每个主要页面都回答同一个问题：
    - 现在在做什么
    - 做完后去哪里
    - 如果卡住该回哪一页
- 完成标准：
  - 用户不翻日志、不懂技术，也能看懂“现在到哪了”和“下一步去哪”。

##### 第六阶段-B：场景解释并回
- 要解决的本质问题：
  - 用户看到这一场写出来了，但不一定一眼明白“这一场在推进什么、为什么这样写”。
- 这一步只做什么：
  - 给每场结果补一层极轻的人话解释。
  - 解释只允许回答 3 个问题：
    - 这一场在推进什么
    - 这一场付出了什么代价
    - 为什么下一场必须接着看
- 这一步绝对不做什么：
  - 不准新写一套“创作说明书”。
  - 不准出现“系统认为”“模型判断”“本场任务是”这类幕后口吻。
  - 不准为了可解释，反过来指挥正文怎么写。
- 执行动作：
  - 只从现有真相里翻译：
    - 当前粗纲
    - 当前详纲
    - 当前这场正文
    - 当前账本里已经存在的承接 / 代价 / 钩子
  - 如果现有真相里没有，就宁可不显示，也不准现编。
  - 解释展示必须是附属层，不能反过来掌权。
- 完成标准：
  - 用户能看懂“这场戏为什么这样写”，但仓库里没有长出第二套创作口。

##### 第六阶段-C：静默守卫并回
- 要解决的本质问题：
  - 这轮好不容易压回来的口径，后面最容易因为临时补丁又长回第二口子。
- 这一步只做什么：
  - 把这轮已经验证过的规则继续锁进唯一主链。
  - 前台只显示人话结果，不自己再做一遍裁判。
  - 把“回哪一页补”的提示继续统一到现有主链翻译层。
- 这一步绝对不做什么：
  - 不准新增重 gate。
  - 不准新增命中词校验。
  - 不准新增正文补丁链。
  - 不准为了防回弹，再造一层平行判断逻辑。
- 执行动作：
  - 复核当前“状态展示”“场景解释”“阻塞提示”是不是都只吃现有主链结果。
  - 发现任何页面自己另算 ready / blocked / 下一步，一律回主链。
  - 守卫如果需要新增，只能新增在唯一真相源那条线上，不能散落到前台各页。
- 完成标准：
  - 用户看见的是更清楚的主路，不是更多规则。
  - 工程上没有反弹回第二写口、第二发布口、第二裁判口。

#### 第六阶段验收清单
- 必须同时成立：
  1. 用户能一眼看懂“整个项目到哪一步了”。
  2. 用户能一眼看懂“当前这一批 / 这一场正在干嘛”。
  3. 用户能看懂“这场戏为什么这样写”，但解释层没有掌权。
  4. 前台没有重新长出内部词、假进度、第二套判断口。
  5. 这轮并回没有把主线带回重工程、轻创作。

#### 给执行人的硬提醒
- 这一步不是加功能，是把外部学习变成我们自己的收口动作。
- 能复用现有状态、现有真相、现有翻译层，就不准新造。
- 如果你发现自己在写“第二套说明、第二套判断、第二套状态”，立刻停下重做。

### 第七阶段：最终收口
- 目标：
  - 正式判断“剧本正文质量这最后一截”有没有过线。
- 动作：
  - 汇总 3 轮真测结果。
  - 汇总外部样本吸收结果。
  - 重新看一遍：粗纲、人物、详纲有没有回退，正文有没有真正上去。
- 完成标准：
  - 可以正式说：系统已经不只是“能写出来”，而是开始“稳定写出优秀的正文”。

## 本轮实际执行结果（2026-03-19）
- 这份计划的原始目标没有改，后面接力的人仍然按这份计划理解“理想全量完成”是什么。
- 但这一轮真实落地结果要单独写死，防止任务卡和计划再次分叉。
- 本轮已经完成：
  1. 第一阶段到第五阶段完成。
  2. 第六阶段里，6-A 只完成了“进度条修真”和人话化展示，未完成原计划要求的双层状态反馈。
  3. 第六阶段里，6-C 静默守卫并回完成，且没有长出第二裁判口。
  4. 第四阶段 3 轮真测通过，说明正文主路这轮压法已经开始生效。
- 本轮未完成：
  1. 第六阶段-B“场景解释并回”未实现。
  2. 第六阶段-A“当前这一批 / 这一场正在干嘛”的前台展示未实现；不是底座没有，而是底座还没接到前台。
- 所以本轮真实结论不是“原计划 100% 全量完成”，而是：
  1. 这条主线核心目标已阶段性收口。
  2. 当前可按约 85% 判断。
  3. 下一轮只补一个真东西：Phase 6-B 场景解释层。
  4. 双层状态反馈前台接入，不单开新主线，跟着 Phase 6-B 一起补，避免再次拆散主路。

## 这轮下一棒就做什么
1. 先把当前真实结论写进任务卡和 worklog：问题在正文，不在架构。
2. 先做第一阶段：冻结当前真相。
3. 立刻进第二阶段：先压标记泄露、说明感和拖长对白。
4. 外部学习从 `waoo` 开始，但只看产品流程和工程收口，不看“谁更会写剧本”。

## 这份计划绝不接受的假完成
- 只证明“这一场看起来顺一点了”。
- 又把问题说回架构层。
- 学外部项目学成“别人功能更多，所以我们也要加”。
- 正文没真变好，只是把标记藏深一点。
- 用更多限制取代真正的正文升级。

## 最后一句
- 这份计划不是为了证明系统已经很厉害。
- 这份计划是为了把最后那一截最容易丢人的东西真正压过去：让正文不只是能写出来，而是真的开始像一部优秀作品。
