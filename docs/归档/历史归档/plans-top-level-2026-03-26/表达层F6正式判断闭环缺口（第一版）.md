# 表达层F6正式判断闭环缺口（第一版）

更新时间：2026-03-16 08:14

## 这份文档解决什么本质问题
- 不是再说一遍“表达层很重要”。
- 真正要解决的是：为什么 `F6` 明明已经有拆法、有提示词、有样本底板，却还没进入稳定判断链。

## 先说结论
- `F6` 现在最大的缺口，不是创作方向没想法。
- 而是还没有收成“文档标准 -> 审计标准 -> 修补标准 -> postflight 标准 -> 样本封口”的正式闭环。

## 当前已经有的东西
- 已有正式总标准：表达层按 4 项验
- 已有细拆坏法：对白不像人话、三要素不成一体、场景不可演、不好续写
- 已有主生成口径：表达层要求已经压进 prompt
- 已有修补口径：repair 也继承了表达层主路
- 已有样本底板：过/不过样本卡和第一轮复述记录都在

## 为什么还不能叫“进入稳定判断链”

### 一、还没进正式审计问题码
- 现在审计更偏字段完整、事实落地、推进链和信号。
- 还没有 `F6` 专属问题码去稳定承接：
  - 说明味重
  - 人物味弱
  - 嘴感不顺
  - 场景不可演
  - 不好续写

### 二、还没进正式修补映射
- 现在 repair mapping 主要还是接结构类和推进类问题。
- `F6` 的细问题还没有统一问题名，也没有完整映射到修补动作。

### 三、还没进 postflight 收口
- 现在 postflight 还没有用表达层四项去判断“这场是不是虽然生成出来了，但其实不好继续接”。

### 四、还没补齐“好续写”的判断动作
- 对白像人话、场景能演已经拆得比较清楚。
- 但“结果能续写”现在还只是结果词，不是同等级正式判断动作。

### 五、还没钉死 F6 的稳定样本封顶条件
- 现在有样本底板，但还没正式回答：
  - 连续多少笔才算够
  - 要跨哪些位置
  - 要覆盖哪些坏法

## 所以当前最该收的顺序
1. 先把 `F6` 四项收成正式问题码
2. 再把问题码接进 audit / repair / postflight
3. 再补“好续写”判断拆解
4. 最后补 `F6` 稳定通过的样本封顶条件

## 最后结论
- `F6` 当前不是没底子，而是差最后一层“正式判断闭环”。
- 只要这层闭环不立住，表达层就会一直停在“方向对、结果也有好样本，但还不能正式稳定承诺”。
