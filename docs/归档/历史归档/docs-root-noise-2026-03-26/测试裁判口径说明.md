# 测试裁判口径说明

更新时间：2026-03-22

## 当前结论

- `tools/e2e/` 里旧的真实链路、seeded、长测、观察器脚本已整体退役并删除。
- 删除原因不是“偶发不稳定”，而是这批脚本长期混用 UI 黑盒、内部 API 旁路、直接改 `projects.json`、复用历史 `userdata-*`、旧字段旧术语裁判，结论结构性不可信。
- 当前仓库不再允许把 `tools/e2e/` 当正式产品验收入口。

## 当前只保留的工具

- `tools/e2e/e2e-output.mjs`
  - 只负责清理和准备 `tools/e2e/out` 输出目录。
- `tools/e2e/electron_launch_smoke.mjs`
  - 只验证 Electron 壳能否启动。
- `tools/e2e/contract_guard_check.mjs`
  - 只做静态合同守卫，不跑 UI 主链。
- `tools/e2e/run_dev_mock.ps1`
  - 只做本地开发辅助。

## 正式验收规则

- 正式产品验收不再使用任何已删除的 `tools/e2e/*.mjs` 历史脚本。
- 需要恢复 E2E 时，必须从零重建。
- 新脚本必须同时满足 5 条：
  - 只走真实 UI 主链，不允许 `page.evaluate()` 旁路内部生成 API。
  - 不允许直接改 `projects.json`、不允许脚本造状态。
  - 不允许默认绑定历史 `tools/e2e/out/userdata-*`。
  - 只认当前正式合同和当前字段，不认旧术语、旧状态字符串、旧兼容字段。
  - 结果必须区分“产品失败”“采样失败”“环境失败”，禁止吞错后硬判产品失败。

## 当前执行口径

- 需要最小烟测时，只允许运行 `electron_launch_smoke.mjs`。
- 需要静态门禁时，只允许运行 `contract_guard_check.mjs`。
- 需要真正的产品验收时，走新测试体系，不能回捞已删除脚本。
