# 从源头解决剧本质量五项问题——全层修复计划

## 五个问题的根因和落点

| #   | 问题         | 根因层                                            | 要改的文件                           |
| --- | ------------ | ------------------------------------------------- | ------------------------------------ |
| P1  | 第4-5集重复  | 详纲层：相邻集 sceneByScene 地点+动作重叠         | `generation-stage-prompts.ts`        |
| P2  | 第7集发胖    | 剧本主生成层：场级预算只出现在 repair，主生成没带 | `create-script-generation-prompt.ts` |
| P3  | 人物后期扁平 | compact 压缩层：hiddenPressure/fear 被压没        | `compact-script-context.ts`          |
| P4  | 对白后期退化 | compact 压缩层：同 P3                             | `compact-script-context.ts`          |
| P5  | 第8-10集偏瘦 | 详纲层：拉扯只给一句话，骨架太薄                  | `generation-stage-prompts.ts`        |

## 改动计划

### 改动 1：详纲 prompt 加去重规则 + 拉扯最低密度（P1 + P5）

**文件**：`src/main/application/workspace/generation-stage-prompts.ts`

**改动内容**：在 `buildDetailedOutlineActPrompt` 的 prompt 规则数组中新增两条：

1. **去重规则**：`相邻两集的 sceneByScene 不准出现相同地点+相同动作组合；如果上一集已经在"某地点抢某物"，下一集的 setup 或 tension 就不准再写"在同一个地点抢同一件物"——必须换地点、换目标、换手法或换阶段（从抢变成藏、从藏变成换、从换变成毁）。`

2. **拉扯最低密度规则**：`sceneByScene 的 tension 不能只写一句话的动作指令（如"黎明按住残党逼问"）；至少要写清"我方动作→对方回应→局面变化"三步。如果 tension 只有一句，说明这场骨架太薄，必须补到两步以上。`

**不改什么**：不改 `generate-detailed-outline.ts` 的执行逻辑，只改 prompt。

### 改动 2：场级预算数字前移到主生成 prompt（P2）

**文件**：`src/main/application/script-generation/prompt/create-script-generation-prompt.ts`

**改动内容**：在主生成 prompt 的 `buildCurrentEpisodeTaskLines` 函数（或同文件的 prompt 组装处），新增一条场级预算指令：

```
当前集预算：
- 2 场集：每场目标 420-560 字，超过 650 字算写胖
- 3 场集：每场目标 280-380 字，超过 450 字算写胖
- 4 场集：每场目标 220-300 字，超过 340 字算写胖
当前总预算：2 场集 950-1150 字，3 场集 950-1150 字，4 场集 950-1150 字
超过整集上限的场先收，不准一场写胖再靠后面场来补
```

这些数字当前只在 `build-episode-scene-directives.ts` 的 repair 分支里出现。要前移到主生成 prompt 里，让模型第一次写时就按预算写。

**关联文件**：`build-episode-scene-directives.ts` 里对应的场级预算数字保持不变（repair 阶段仍然用），但主生成阶段也要同步带上。

### 改动 3：compact 模式保留 hiddenPressure + fear（P3 + P4）

**文件**：`src/main/application/script-generation/prompt/compact-script-context.ts`

**改动内容**：`buildCompactedCharacterBlock` 函数中，compact 模式（`maxChars <= 1600`）下的人物行，从当前的：

```
name + publicMask(22字) + conflictTrigger(18字) + goal(18字)
```

改成：

```
name + hiddenPressure(22字) + fear(18字) + conflictTrigger(18字) + goal(18字)
```

**逻辑**：

- 砍掉 `publicMask`（"表面怎么演"在 compact 下不如"暗里卡着什么"重要）
- 保住 `hiddenPressure`（这是写人物层次的核心锚点——人物内心的撕裂）
- 保住 `fear`（这是写人物代价和选择的核心锚点——最怕什么决定他怎么选）
- `conflictTrigger` 和 `goal` 继续保留

总字数基本不变（砍一个加两个，但 fear 和 hiddenPressure 都有 18-22 字限制），不会撑爆 compact 预算。

### 改动 4：VO 正则修复 bug（附带发现）

**文件**：`src/main/application/script-generation/runtime/repair-generated-scenes.ts`

**改动内容**：`repairVoiceOver` 函数的正则 bug。当前正则：

```js
;/([^：:\n]+)：\s*（画外音）\s*）([^(\n]+)/g
```

要求格式是 `角色名：（画外音））对白`（多一个右括号），但实际模型输出是 `角色名：（画外音）对白`（没有多余右括号）。

修正：把所有 5 条正则里的 `\s*）` 去掉，改成：

```js
;/([^：:\n]+)：\s*（画外音）\s*([^(\n]+)/g
```

## 不做的事

1. **不修钩子弱**——这是模型创作能力上限，不是 prompt 或代码能修的。
2. **不修详纲生成逻辑**——只改 prompt，不改 `generate-detailed-outline.ts` 的代码。
3. **不修 repair 流程**——上一轮已经修好了修复顺序和 guard。
4. **不加新的兜底机制**——所有改动都是从源头给模型更好的指令，不是事后补丁。

## 改动顺序

1. 先改 compact（P3+P4）——改动最小，只动一个函数
2. 再改 VO 正则 bug——改动最小，只动 5 条正则
3. 再改详纲 prompt（P1+P5）——只加规则，不改逻辑
4. 最后改剧本主生成 prompt（P2）——需要看清 prompt 组装位置

## 验证方式

每步改完：

1. `npm run typecheck:node`
2. `npm run build`
3. 相关板块测试全跑一遍

全部改完后：4. 跑真实 10 集 API 全链验证 5. 对比 evidence：字数分布、VO 数量、人物层次、相邻集去重
