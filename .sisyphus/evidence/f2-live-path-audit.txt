## Legacy Code Inspection

### startScriptGeneration (application layer, NOT IPC entry)
- **Location**: `src/main/application/script-generation/start-script-generation.ts`
- **Status**: NOT an IPC handler - it's an internal application function
- **Used by**: Worker (`script-generation-worker.ts:55`)
- **NOT reachable from renderer** - renderer only calls IPC handlers
- **Worker infrastructure exists but is disconnected**:
  - `runScriptGenerationInWorker()` exists but is never called by orchestrator
  - Orchestrator's `runScriptGenerationBatch` option is injected as direct function (line 83-84 of handlers.ts)
  - Worker thread bootstraps via `src/main/index.ts:74-76` but workerData kind='script_generation' is never posted

**Conclusion**: Legacy `startScriptGeneration` and worker are DEAD CODE, not bypass paths.

### stopScriptGenerationRun (registry)
- **Location**: `src/main/application/script-generation/script-generation-run-registry.ts:28`
- **Purpose**: Legacy worker termination
- **Called by**: Stop IPC handler fallback (line 341) only when `activeOrchestrators.get(projectId)` returns falsy
- **Normal operation**: Never called because orchestrator is always registered before execute() and deleted after
