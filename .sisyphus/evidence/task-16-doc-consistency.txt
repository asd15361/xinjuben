### 1. docs/system-authority（系统定义权与主权规则）.md
Added new section "Authority Constitution & Error Semantics" at end of file:

**Authority owned facts:**
- stage, generation status, resume eligibility, blocked reason
- These four types can ONLY be produced by main/orchestrator
- Renderer only consumes, never produces business truth

**Forbidden patterns:**
- `catch { setStage(...) }` - cannot use catch to secretly set stage
- `if (!result) setStage(...)` - cannot use local judgment to forge stage
- `optimistic core updates` - cannot optimistically estimate core state
- `local interpretation overrides authority` - cannot use local interpretation to override authority declaration

**Error rule:**
- Authority failure = explicit error, NOT fallback, NOT progress, NOT silent ignore
- Required: explicit error surface → block progress → record reason

**Legacy compat:**
- migrate success → continue main flow
- migrate fail → block + explicit error
