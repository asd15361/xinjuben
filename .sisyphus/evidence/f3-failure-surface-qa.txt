## Key Implementation Patterns Confirmed

### Pattern 1: Error Kind Enforcement
All authority failures MUST be `kind: 'error'`. No warnings or info notices.
```typescript

  source: 'system',
  kind: 'error',  // Always error - never warning or info
  title: display.title,

  ...
}
```

### Pattern 2: No Automatic Stage Progression
Recovery actions are restricted:
- `not_recoverable` → `undefined` (no action)
- `manual_retry` → `undefined` (user must manually retry)
- `refresh_project` → Points to 'chat' reload, NOT to blocked stage
- `reload_workspace` → Points to 'chat'
- `fix_contract_input` → `undefined` (user must fix)

### Pattern 3: Constitution Violations Always Block
