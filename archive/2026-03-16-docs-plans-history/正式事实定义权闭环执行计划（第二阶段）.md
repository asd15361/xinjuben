# 正式事实定义权闭环执行计划（第二阶段）

## 目标

把“正式事实定义权”从第一版 contract 和前端声明区，推进成真正的主进程受控闭环。

## 第二阶段顺序

1. 新增 `main/application/formal-fact/*`
   - `declare-formal-fact.ts`
   - `confirm-formal-fact.ts`
   - `validate-formal-fact.ts`
   - `evaluate-formal-fact-elevation.ts`
2. 新增 `ipc/workflow/formal-fact-handlers.ts`
3. 新增 preload `workflow` 正式事实 API
4. 把粗纲页声明动作切到主进程
5. 限制其他工序只能读取 `confirmed formal facts`

## 验收标准

- renderer 不再直接决定正式事实是否进入主链
- 正式事实声明动作必须走 `main/application`
- 只有 `outline` 工序允许声明 / 确认正式事实
- 其他工序只能消费 `confirmed formal facts`
