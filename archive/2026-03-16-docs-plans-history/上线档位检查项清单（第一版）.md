# 上线档位检查项清单（第一版）

## 一、这份文档解决什么问题

这份文档只回答一件事：

**三档上线口径，到底怎么一条一条检查。**

不是继续讲概念。  
而是把“属于哪一档”变成能逐条打勾的检查项。

---

## 二、使用方法

以后每次判断项目属于哪一档，不准只凭感觉。

固定做法：

1. 从 `外部正式上线` 往下看
2. 哪一档有检查项没过，就降一档
3. 最后停在哪一档，就按哪一档汇报

---

## 三、第一档：内部自用检查项

只要下面 6 条全过，就算达到 `内部自用`。

### I1. build 能稳定通过

检查标准：

- 当前 build 能通过

### I2. 主链能完整走通

检查标准：

- 从聊天入口到下游主链能跑通

### I3. Chat-first 入口成立

检查标准：

- 用户进入项目后默认就是聊天入口
- 不再把表单当入口本体

### I4. confirmed facts 能稳定进入下游

检查标准：

- 正式事实确认后，下游可以稳定消费

### I5. 剧本 Gate 能稳定工作

检查标准：

- 不合格输入会被挡住
- 合格输入能进入后续链路

### I6. 失败 / 续跑链路有真实验证

检查标准：

- 至少有一版失败 / 续跑验证证据

当前判断：

- 已通过

---

## 四、第二档：小范围内测检查项

只要下面 5 条全过，就算达到 `小范围内测`。

前提：

- 第一档全部通过

### T1. 主要入口对真实用户可理解

检查标准：

- 用户不需要懂内部结构，也知道从哪里开始

### T2. 主链中的“能做什么 / 不能做什么”足够清楚

检查标准：

- 下游约束不是藏着的
- 用户看得见当前为什么能继续、为什么不能继续

### T3. 当前已知问题有正式记录

检查标准：

- 当前仍成立的问题已经写成正式文档

### T4. 系统边界有正式说法

检查标准：

- 当前阶段怎么说
- 当前属于哪一档
- 为什么还不能正式上线

这 3 件事都已经形成现行口径

### T5. 剧本主要问题已经能被主链捕捉

检查标准：

- 虽然质量还没稳定封顶
- 但主要问题已经能被 gate / audit / repair 看见

当前判断：

- 已基本通过

当前还差：

- 不能放大范围
- 还要继续补质量样本和更细门禁

---

## 五、第三档：外部正式上线检查项

只要下面 7 条全过，才算达到 `外部正式上线`。

前提：

- 第二档全部通过

### O1. 当前阶段和能力边界已完全收口

检查标准：

- 现行总计划、任务卡、上线文档口径稳定一致

### O2. 好剧本的正式质量维度已经明确

检查标准：

- 不是凭感觉判断“写得好”
- 已有正式质量维度

### O3. 有一批可复盘的样本级质量证明

检查标准：

- 能看到样本
- 能看到问题
- 能看到修补结果
- 能看到最终判断

### O4. 更深连续性与修补闭环已经收死

检查标准：

- continuity / tone / hook 承接进入更稳定闭环

### O5. 架构纪律达到成熟态

检查标准：

- 关键大文件和职责混层已继续下降
- 仓库不会轻易重新长歪

### O6. 性能与体验不再明显拖主链后腿

检查标准：

- 关键路径已经足够顺
- 用户不需要大量解释也能走通

### O7. 当前外部正式上线阻断项全部解除

检查标准：

- `B1-B5` 已逐条解除

当前判断：

- 未通过

当前最核心没过项：

- `O2`
- `O3`
- `O4`
- `O5`
- `O6`
- `O7`
