# 旧项目剧本执行计划与续跑机制学习报告（第一版）

## 一、核心结论

旧项目在剧本生成阶段最值钱的，不是“跑了一串流程”，而是把以下三份状态对象做成了真正的中枢：

1. `ScriptGenerationExecutionPlan`
2. `ProjectProgressBoard / ResumeResolution`
3. `ScriptGenerationFailureResolution`

换句话说，旧项目之所以能续跑、能局部重写、能失败恢复，不是因为页面里塞了很多按钮，而是因为这些状态对象在背后持续约束生成链。

## 二、三份最值得迁移的状态对象

### 1. ScriptGenerationExecutionPlan

#### 作用
- 定义本次生成是首次跑、续跑还是重写
- 计算从哪一集开始
- 定义目标集数、前缀锁定范围、运行档位
- 把本次生成任务拆成一组可执行条目

#### 第一版最小字段建议
- `mode`
- `rewriteStartEpisode`
- `targetEpisodeCount`
- `seasonTotalEpisodes`
- `baseCompletedEpisodes`
- `generationOutline[].episode`
- `generationOutline[].title`
- `generationOutline[].content`
- `lockedPrefixScripts[].episode`
- `runtimeProfile.batchSize`
- `runtimeProfile.runtimeStrictness`

#### 旧项目关键文件
- `D:\project\juben\src\renderer\src\services\scriptGenerationExecutionPlanService.ts`
- `D:\project\juben\src\renderer\src\services\scriptGenerationInputContractService.ts`
- `D:\project\juben\src\renderer\src\services\scriptGenerationRuntimeProfileService.ts`

### 2. ProjectProgressBoard / ResumeResolution

#### 作用
- 判断当前项目能不能续跑
- 记录当前批次处于暂停、失败还是执行中
- 给出从哪一集继续、为什么从这里继续

#### 第一版最小字段建议
- `episodeStatuses[].episode`
- `episodeStatuses[].status`
- `episodeStatuses[].batchIndex`
- `episodeStatuses[].reason`
- `batchContext.status`
- `batchContext.startEpisode`
- `batchContext.endEpisode`
- `batchContext.resumeFromEpisode`
- `batchContext.currentBatchIndex`
- `batchContext.batchSize`
- `batchContext.stageContractFingerprint`
- `batchContext.updatedAt`

#### 旧项目关键文件
- `D:\project\juben\src\renderer\src\services\scriptGenerationResumeService.ts`
- `D:\project\juben\src\renderer\src\services\projectService.ts`
- `D:\project\juben\src\renderer\src\services\generationService.ts`

### 3. ScriptGenerationFailureResolution

#### 作用
- 区分“可重试失败、手动停止、真实失败”
- 失败时返回进度板，而不是只弹一个错误
- 给锁恢复和自动回退留正式落点

#### 第一版最小字段建议
- `kind`
- `reason` / `errorMessage`
- `board`
- `eventId`
- `lockRecoveryAttempted`

#### 旧项目关键文件
- `D:\project\juben\src\renderer\src\services\scriptGenerationFailureService.ts`
- `D:\project\juben\src\renderer\src\services\scriptGenerationLockRecoveryService.ts`
- `D:\project\juben\src\renderer\src\services\generationService.ts`

## 三、对新项目的直接启发

### 1. 先统一状态对象，再写具体执行流

新项目第一版不要急着把“真正批量生成”全部补出来。  
更重要的是先把下面三份对象做成共享合同：

- `ExecutionPlan`
- `ProgressBoard`
- `FailureResolution`

只要这三份对象先统一，后面的续跑、失败回退、局部重写才不会各算各的。

### 2. 当前新项目已经迈出的第一步

本轮新项目已经先落了：

- 第一版 `script-generation` 合同
- 第一版生成前 gate
- 第一版执行计划构建器
- 剧本页的执行计划可视化

这意味着新项目已经从“只会提醒缺什么”，升级到“已经会算这次该怎么跑”。

## 四、接下来最该做什么

建议按这个顺序继续：

1. 补 `ProjectProgressBoard`
2. 补 `ResumeResolution`
3. 补 `FailureResolution`
4. 再把真正的剧本批次执行器接进来

这样能最大程度继承旧项目的强点，同时避免把旧项目的大耦合原样抄进来。
