# 本地持久化方案设计

## 目标

- **内容数据本地化**：剧本正文、草稿、运行时状态写本地，不依赖 PB
- **元数据索引保留 PB**：账号、积分、项目列表、权限继续用 PB
- **按用户和项目隔离**：`userId/projectId` 目录组织
- **支持启动恢复**：应用重启后能读取上次状态
- **支持保存草稿/运行时状态**：生成中途可暂停、可恢复
- **修复/重写后落本地**：内容更新只写本地

---

## 方案选型

### 方案 A：复用现有 `project-store`（JSON 文件）

**现状：**
- 位置：`app.getPath('userData')/workspace/projects.json`
- 结构：单文件存储所有项目完整快照
- 优点：已有代码、支持原子写、有迁移机制
- 缺点：单文件可能变大、无查询能力

**适配改造：**
```typescript
// 保留：元数据仍写 PB
interface ProjectMetadata {
  id: string
  userId: string
  name: string
  workflowType: string
  stage: string
  genre: string
  updatedAt: string
}

// 新增：内容数据本地化
interface LocalProjectContent {
  projectId: string
  scriptDraft: ScriptSegmentDto[]      // 剧本正文
  chatMessages: ChatMessageDto[]       // 聊天历史
  outlineDraft: OutlineDraftDto | null // 大纲（可选保留 PB）
  characterDrafts: CharacterDraftDto[] // 人物（可选保留 PB）

  // 运行时状态（可选持久化）

```

**存储路径：**
```
userData/
  workspace/
    projects.json           # 元数据索引（轻量，同步 PB）
    content/
      {userId}/
        {projectId}/
          content.json      # 完整内容快照
          scripts/          # 分集存储（可选优化）
            episode-1.json
            episode-2.json
          runtime.json      # 运行时状态（可清理）
```

### 方案 B：SQLite 本地数据库

**优点：**
- 查询能力强（按用户/项目/集数筛选）
- 支持增量更新
- 可加索引优化性能

**缺点：**
- 需要引入 `better-sqlite3`
- 增加部署复杂度
- 现有代码需重构

**表结构：**
```sql
-- 内容存储
CREATE TABLE local_content (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  project_id TEXT NOT NULL,
  content_type TEXT NOT NULL, -- 'script_draft' | 'chat_messages' | 'outline' | 'characters'
  content_json TEXT NOT NULL,
  version INTEGER NOT NULL,
  updated_at TEXT NOT NULL,
  UNIQUE(user_id, project_id, content_type)
);

-- 运行时状态（可清理）
CREATE TABLE runtime_state (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  project_id TEXT NOT NULL,
  state_type TEXT NOT NULL, -- 'progress_board' | 'failure_resolution' | 'state_ledger'
  state_json TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  UNIQUE(user_id, project_id, state_type)
);

-- 索引
CREATE INDEX idx_content_user_project ON local_content(user_id, project_id);
CREATE INDEX idx_runtime_user_project ON runtime_state(user_id, project_id);
```

### 方案 C：混合方案（推荐）

**短期（1-2 周）：** 复用现有 `project-store` JSON 文件
- 最小改动，快速落地
- 内容数据从 PB 迁出到本地 JSON
- PB 只保留 metadata 索引

**中期（1-2 月）：** 按需引入 SQLite
- 如果单文件超过 10MB 或性能下降
- 迁移到 SQLite，保持接口不变

### 与现有代码集成

```typescript
// 新增：src/main/infrastructure/storage/local-content-store.ts
import { readFile, writeFile, mkdir } from 'fs/promises'
import { join } from 'path'
import { app } from 'electron'

function getContentRoot(userId: string, projectId: string): string {
  return join(app.getPath('userData'), 'workspace', 'content', userId, projectId)
}

export async function readLocalContent(
  userId: string,
  projectId: string
): Promise<ProjectSnapshotDto | null> {
  const contentPath = join(getContentRoot(userId, projectId), 'content.json')
  try {
    const content = await readFile(contentPath, 'utf8')
    return JSON.parse(content)
  } catch {
    return null
  }
}

export async function writeLocalContent(
  userId: string,
  projectId: string,
  content: Partial<ProjectSnapshotDto>
): Promise<void> {
  const root = getContentRoot(userId, projectId)
  await mkdir(root, { recursive: true })
  
  const contentPath = join(root, 'content.json')
  const payload = JSON.stringify(content, null, 2)
  await writeFile(contentPath, payload, 'utf8')
}

// 运行时状态单独存储（可清理）
export async function writeLocalRuntimeState(
  userId: string,
  projectId: string,
  state: {
    scriptProgressBoard?: unknown
    scriptFailureResolution?: unknown
    scriptStateLedger?: unknown
  }
): Promise<void> {
  const root = getContentRoot(userId, projectId)
  await mkdir(root, { recursive: true })
  
  const runtimePath = join(root, 'runtime.json')
  const payload = JSON.stringify(state, null, 2)
  await writeFile(runtimePath, payload, 'utf8')
}
```

## 链路调整

### 当前链路（PB 写）

```
renderer --HTTP--> server --auth/integral--> AI
                |
                v
         pb.collection('project_scripts').update(...)  ❌ 写大 JSON
```

### 新链路（本地写）

```
renderer --HTTP--> server --auth/integral--> AI
  |                    |
  v                    v
本地 content store    返回结果（不含 PB 写）
```

**流程：**
1. renderer 发起生成请求到 server
2. server 鉴权、扣积分、调用 AI
3. server 返回生成结果（不写 PB）
4. renderer 收到结果后写本地 `content.json`
5. server 只写 PB metadata（projectId、updatedAt、statusSummary）

## 迁移步骤

### Step 1: 创建本地存储模块
- 文件：`src/main/infrastructure/storage/local-content-store.ts`
- 功能：`readLocalContent` / `writeLocalContent` / `writeLocalRuntimeState`

### Step 2: 调整 server 返回
- 文件：`server/src/api/routes/scripts.ts`
- 改动：`start` 接口返回生成结果，不再调用 `persistBoard` 写 PB

### Step 3: 调整 renderer 消费
- 文件：`src/renderer/features/script-generation/`
- 改动：收到结果后调用 `writeLocalContent` 而非等 PB 同步

### Step 4: PB Schema 降级
- 文件：`pb_migrations/1745385600_*.js`
- 改动：标记为"兼容迁移"，不再作为长期依赖

### Step 5: 验证
- 生成 >100KB 剧本，确认本地保存成功
- 关闭重开，确认能恢复
- 检查 PB 无完整剧本文本
