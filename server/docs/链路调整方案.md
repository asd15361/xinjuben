# 链路调整方案：server 只鉴权/扣积分，renderer 写本地存储

## 当前链路（PB 写）

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  renderer   │────▶│    server   │────▶│  PocketBase │
│ (发起生成)  │     │(鉴权/扣积分)│     │ (写大 JSON)  │
└─────────────┘     └─────────────┘     └─────────────┘
                           │
                           ▼
                     ┌─────────────┐
                     │  AI Provider│
                     │ (DeepSeek)  │
                     └─────────────┘
```

**问题：**
- server 调用 `persistBoard` 写入完整剧本到 PB（`scriptDraftJson`）
- PB text 字段有 max=5000 限制（已扩容到 100000，但这是临时方案）
- 桌面软件的内容数据应该本地持久化，不是云存储

---

## 新链路（本地写）

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  renderer   │────▶│    server   │────▶│  AI Provider│
│ (发起生成)  │     │(鉴权/扣积分)│     │ (DeepSeek)  │
└─────────────┘     └─────────────┘     └─────────────┘
       │                                        │
       │                                        ▼
       │                                  返回结果
       │                                        │
       ▼◄───────────────────────────────────────┘
┌─────────────┐
│ 本地存储     │
│ (content.json) │
└─────────────┘
```

**流程：**
1. renderer 发起生成请求到 server（`/api/script-generation/start`）
2. server 鉴权、扣积分、调用 AI
3. server 返回生成结果（不写 PB）
4. renderer 收到结果后写本地 `content.json`
5. server 只写 PB metadata（projectId、updatedAt、statusSummary）

---

### 3. renderer 端：接收结果后写本地

**文件：** `src/renderer/features/script-generation/useScriptGeneration.ts`（或对应 hook）

**新增本地写入：**
```typescript
import { writeLocalContent } from '../../main/infrastructure/storage/local-content-store'

async function handleGenerationComplete(result: ScriptGenerationResult) {
  // 写本地 content store
  await writeLocalContent(userId, projectId, {
    scriptDraft: result.generatedScenes,
    scriptProgressBoard: result.board,
    scriptFailureResolution: result.failure,
    scriptStateLedger: result.ledger
  })
  
  // 更新 UI 状态

  setGenerationBoard(result.board)
}
```

### 4. 新增：本地存储模块

**文件：** `src/main/infrastructure/storage/local-content-store.ts`

```typescript
import { readFile, writeFile, mkdir } from 'fs/promises'
import { join } from 'path'
import { app } from 'electron'
import type { ProjectSnapshotDto } from '../../../shared/contracts/project'

function getContentRoot(userId: string, projectId: string): string {
  return join(app.getPath('userData'), 'workspace', 'content', userId, projectId)
}

export async function readLocalContent(
  userId: string,
  projectId: string
): Promise<Partial<ProjectSnapshotDto> | null> {
  const contentPath = join(getContentRoot(userId, projectId), 'content.json')
  try {
    const content = await readFile(contentPath, 'utf8')
    return JSON.parse(content)
  } catch {
    return null
  }
}

export async function writeLocalContent(
  userId: string,
  projectId: string,
  content: Partial<ProjectSnapshotDto>
): Promise<void> {
  const root = getContentRoot(userId, projectId)
  await mkdir(root, { recursive: true })
  
  const contentPath = join(root, 'content.json')
  const payload = JSON.stringify(content, null, 2)
  await writeFile(contentPath, payload, 'utf8')
}

export async function writeLocalRuntimeState(
  userId: string,
  projectId: string,
  state: {
    scriptProgressBoard?: unknown
    scriptFailureResolution?: unknown
    scriptStateLedger?: unknown
  }
): Promise<void> {
  const root = getContentRoot(userId, projectId)
  await mkdir(root, { recursive: true })
  
  const runtimePath = join(root, 'runtime.json')
  const payload = JSON.stringify(state, null, 2)
  await writeFile(runtimePath, payload, 'utf8')
}
```

## PB 只写 metadata

**保留字段：**
```typescript
interface ProjectMetadata {
  id: string
  userId: string
  name: string
  workflowType: string
  stage: string
  genre: string
  updatedAt: string
  generationStatus?: {
    status: 'running' | 'completed' | 'failed'
    completedEpisodes: number
    totalEpisodes: number
  }
  visibleResult?: {
    status: 'success' | 'failed' | 'pending'
    description: string
  }
}
```

**不再写：**
- `scriptDraftJson`（完整剧本正文）
- `scriptProgressBoardJson`（运行时进度板）
- `scriptFailureResolutionJson`（失败解决方案）
- `scriptRuntimeFailureHistoryJson`（失败历史）
- `scriptStateLedgerJson`（状态分类账）

4. ✅ 修复/重写后本地内容更新
