# 内容本地真相源迁移报告

## 迁移背景

上一条主线（scriptDraftJson 空值修复）虽然完成了根因修复和回归测试，但暴露了一个架构问题：**我们错误地将桌面创作软件的内容数据（剧本正文）存储在云端 PocketBase 中**，这导致了：

1. **schema 限制问题**：PB text 字段默认 max=5000，需要手动扩容
2. **网络依赖**：桌面软件需要联网才能保存创作内容
3. **性能问题**：大 JSON 写入慢，容易超时
4. **架构错位**：这是桌面创作软件，不是云文档系统

## 新架构口径

| 职责 | PocketBase | 本地存储 |
|------|------------|----------|
| 账号 | ✅ 唯一真相源 | ❌ 不存储 |
| 登录 | ✅ 唯一真相源 | ❌ 不存储 |
| 积分 | ✅ 唯一真相源 | ❌ 不存储 |
| 订单 | ✅ 唯一真相源 | ❌ 不存储 |
| 权限 | ✅ 唯一真相源 | ❌ 不存储 |
| 项目索引 | ✅ 轻量 metadata | ❌ 不存储 |
| 剧本正文 | ❌ 不存储 | ✅ 唯一真相源 |
| 草稿正文 | ❌ 不存储 | ✅ 唯一真相源 |
| 运行时状态 | ❌ 不存储 | ✅ 唯一真相源 |
| 聊天历史 | ❌ 不存储 | ✅ 唯一真相源 |

**核心原则：**
- PocketBase 是账户与计费真相源（账号、登录、积分、订单、权限、项目索引）
- 本地项目存储是内容真相源（剧本正文、草稿、运行时状态）
- 这是桌面创作软件，不是云文档系统：用户内容优先本地持久化，PB 只管账号和钱

### 2. 盘点 PB 剧本相关字段 ✅

**文件：** `server/docs/PB 剧本字段盘点.md`

**必须退出 PB 的字段（高优先级）：**
1. `project_scripts.scriptDraftJson` - 剧本草稿正文（单集 5-15KB，全集 100-500KB+）
2. `project_scripts.scriptProgressBoardJson` - 生成进度板（运行时状态）
3. `project_scripts.scriptRuntimeFailureHistoryJson` - 失败历史（调试日志）
4. `project_scripts.scriptStateLedgerJson` - 状态分类账（运行时状态）

**可保留在 PB 的字段（元数据/索引）：**
- `projects.storyIntentJson` - 核心元数据（<5KB）
- `projects.entityStoreJson` - 人物/势力索引（<50KB）
- `project_outlines.outlineDraftJson` - 大纲（结构化元数据，<100KB）
- `project_characters.*` - 人物卡片索引（<50KB）

### 4. 设计链路调整方案 ✅

**文件：** `server/docs/链路调整方案.md`

**当前链路（PB 写）：**
```
renderer --HTTP--> server --auth/integral--> AI
                |
                v
         pb.collection('project_scripts').update(...)  ❌ 写大 JSON
```

**新链路（本地写）：**
```
renderer --HTTP--> server --auth/integral--> AI
  |                    |
  v                    v
本地 content store    返回结果（不含 PB 写）
```

### 5. 创建验证脚本 ✅

**文件：** `src/main/infrastructure/storage/verify-local-persistence.test.ts`

**验收标准：**
- 生成 >100KB 剧本（20 集，每集 5-10KB）
- 本地 `content.json` 成功保存
- 关闭重开能恢复
- 修复/重写后内容更新
- PB 无完整文本
